Security Bulletin

Microsoft Security Bulletin MS09-070 - Importante
=================================================

Vulnerabilidades nos Serviços de Federação do Active Directory podem permitir a execução remota de código (971726)
------------------------------------------------------------------------------------------------------------------

Publicado: terça-feira, 8 de dezembro de 2009 | Atualizado: quarta-feira, 9 de dezembro de 2009

**Versão:** 1.1

### Informações Gerais

#### Sinopse

Esta atualização de segurança elimina duas vulnerabilidades relatadas em particular no Microsoft Windows. A mais grave dessas vulnerabilidades poderá permitir a execução remota de código se um invasor enviar uma solicitação HTTP especialmente criada a um servidor Web com ADFS habilitado. O invasor precisaria ser um usuário autenticado para explorar qualquer uma dessas vulnerabilidades.

Esta atualização de segurança foi classificada como Importante para edições afetadas do Windows Server 2003, Windows Server 2003 x64 Edition, Windows Server 2008 e Windows Server 2008 x64 Edition. Para obter mais informação, consulte a subseção **Software afetado e não afetado** nesta seção.

A atualização de segurança elimina as vulnerabilidades corrigindo a validação que servidores Web com ADFS habilitado aplicam a cabeçalhos de solicitação enviados por um cliente da Web. Para obter mais informação sobre as vulnerabilidades, consulte a subseção Perguntas frequentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.**  A maioria dos clientes tem o recurso de atualizações automáticas habilitado e não precisará tomar nenhuma providência porque esta atualização de segurança será baixada e instalada automaticamente. Os clientes que não têm o recurso de atualizações automáticas habilitado precisarão verificar as atualizações e instalar esta atualização manualmente. Para obter informações sobre opções de configuração específicas em atualizações automáticas, consulte o [Artigo 294871 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Consulte também a seção **Orientação e ferramentas de detecção e implantação** mais adiante neste boletim.

**O que são os Serviços de Federação do Active Directory?**  
Os Serviços de Federação do Active Directory (ADFS) se baseiam na arquitetura emergente de serviços Web aceita pelo setor, que é definida em especificações WS-\*. O ADFS ajuda a usar o logon único (SSO) para autenticar usuários em vários aplicativos da Web relacionados, durante uma única sessão online. O ADFS faz isso compartilhando com segurança a identidade digital e os direitos através dos limites de segurança e empresariais.

**Como posso verificar se o ADFS está instalado em meu servidor? **
Para verificar se o ADFS está instalado em seu servidor, clique em **Iniciar**, clique em **Painel de Controle**, clique em **Adicionar/Remover Programas**, clique duas vezes em **Componentes do Windows** e procure os **Serviços de Federação** como um componente instalado.

**O que é o Windows Server 2003 R2?**  
O Windows Server 2003 R2 estende o sistema operacional Windows Server 2003, fornecendo uma maneira mais eficiente de gerenciar e controlar o acesso a recursos locais e remotos, ao mesmo tempo em que se integra facilmente ao ambiente do Windows Server 2003 existente. O Windows Server 2003 R2 fornece uma plataforma Web expansível com segurança reforçada, interoperabilidade perfeita com sistemas baseados no UNIX, além de possibilitar novos cenários, inclusive gerenciamento simplificado de servidor de ramificação, gerenciamento aprimorado de identidade e acesso, e gerenciamento de armazenamento mais eficiente. O Windows Server 2003 R2 Datacenter Edition e o Windows Server 2003 R2 Enterprise Edition também proporcionam novo licenciamento dinâmico, o que permite aos clientes obter ainda mais valor da virtualização do servidor.

Para obter mais informações sobre o Windows Server 2003 R2, consulte o site do [Windows Server 2003 R2](http://www.microsoft.com/windowsserver2003/default.mspx).

**O que são especificações WS-\* da Web?**

As especificações para uma Arquitetura de serviços Web baseada em padrões de setor, como SOAP (Simple Object Access Protocol); XML; WSDL (Web Services Description Language) e UDDI (Universal Description, Discovery, and Integration). WS-\* oferece uma base para fornecer soluções comerciais completas e interoperáveis para a empresa estendida, o que inclui a capacidade de gerenciar identidade e segurança federadas.

O modelo de serviços Web se baseia na idéia de que sistemas empresariais são programados em linguagens diferentes, com diferentes modelos de programação, executados em muitos tipos diferentes de dispositivos, pelos quais são acessados. Os serviços Web são um meio de construir sistemas distribuídos que podem se conectar e interagir uns com os outros de maneira fácil e eficiente pela Internet, independentemente da linguagem em que são programados ou da plataforma em que são executados.

Vulnerabilidade de falsificação de logon único no ADFS - CVE-2009-2508  
----------------------------------------------------------------------
  
<span></span>
Uma vulnerabilidade de falsificação nos Serviços de Federação do Active Directory poderá permitir que um invasor represente um usuário autenticado, se o invasor tiver acesso a uma estação de trabalho e ao navegador da Web recentemente usado pelo usuário visado para acessar um site que ofereça logon único.
  
Para exibir esta vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2009-2508](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2508).
  
#### Fatores atenuantes da Vulnerabilidade de falsificação de logon único no ADFS - CVE-2009-2508
  
A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:
  
-   O invasor não pode escolher quem ele deseja representar. O invasor pode representar somente alguém de quem ele tenha conseguido obter um token de autenticação, por exemplo, de um quiosque de Internet.  
-   O ataque é possível somente antes que o tempo limite da sessão se esgote no servidor. Por padrão, as sessões permanecem abertas por 600 minutos no servidor. Se um invasor tentar reutilizar o token de autenticação de uma vítima depois de esgotado esse período de tempo limite, o ataque fracassará.  
-   O invasor precisa ter acesso ao token de autenticação usado pelo usuário anterior no terminal. No caso de um servidor com HTTPs habilitado, esse token de autenticação terá sido protegido em trânsito, e o invasor precisará ter acesso ao computador da vítima (por exemplo, um quiosque de Internet) para explorar a vulnerabilidade. Em um quiosque de Internet, o usuário pode se conectar a um aplicativo Web com logon único (SSO) por ADFS e, em seguida, se desconectar do aplicativo. Um invasor pode então usar esse computador para acessar o aplicativo Web como o usuário anterior, mesmo que o usuário tenha feito logoff.
  
#### Soluções alternativas para a Vulnerabilidade de falsificação de logon único no ADFS - CVE-2009-2508
  
A Microsoft não identificou nenhuma solução alternativa para esta vulnerabilidade.
  
#### Perguntas frequentes sobre a Vulnerabilidade de falsificação de logon único no ADFS - CVE-2009-2508
  
**Qual é o escopo da vulnerabilidade?**    
Essa é uma vulnerabilidade de falsificação. Um invasor que explore esta vulnerabilidade com êxito pode representar outro usuário, o que permite ao invasor obter acesso como o usuário visado a um site que use a funcionalidade de logon único dos Serviços de Federação do Active Directory (ADFS).
  
**O que provoca a vulnerabilidade?**    
Esta vulnerabilidade existe porque falta validação suficiente de gerenciamento de sessão no ADFS.
  
**Para que um invasor pode usar a vulnerabilidade?**    
Em sistemas com acesso compartilhado, como quiosques de Internet, um invasor que explore esta vulnerabilidade com êxito pode acessar recursos pertencentes a outro usuário previamente autenticado. Este ataque por ser executado somente dentro do tempo limite da sessão, configurado pelo administrador de ADFS, que é normalmente configurado como 600 minutos.
  
**De que forma o invasor pode explorar a vulnerabilidade?**    
Um invasor pode explorar esta vulnerabilidade acessando dados específicos do cache do navegador e reutilizando-os para autenticar-se no aplicativo Web que implementa o logon único (SSO) por ADFS. Depois que o tempo limite da sessão se esgota no servidor, o ADFS desativa a sessão autenticada no servidor e não mais permite ao invasor reutilizar essas credenciais.
  
**Quais são os sistemas que mais correm riscos com a vulnerabilidade?**    
Os servidores Web que usam ADFS para fins de logon único são afetados por esta vulnerabilidade. Esta vulnerabilidade não afeta sistemas cliente diretamente.
  
**O que a atualização faz?**    
A atualização elimina a vulnerabilidade descartando corretamente as credenciais de autenticação quando o usuário se desconecta da rede.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre essa vulnerabilidade por meio de divulgação responsável.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**    
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes quando este boletim de segurança foi lançado pela primeira vez.
  
Vulnerabilidade de execução remota de código no ADFS - CVE-2009-2509  
--------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código em implementações dos Serviços de Federação do Active Directory (ADFS). A vulnerabilidade se deve à validação incorreta de cabeçalhos de solicitação quando um usuário autenticado se conecta a um servidor Web com ADFS habilitado. O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado.
  
Para exibir esta vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2009-2509](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2509).
  
#### Fatores atenuantes da Vulnerabilidade de execução remota de código no ADFS - CVE-2009-2509
  
A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:
  
-   O invasor precisa ter credenciais de logon válidas para o servidor vulnerável. Um ataque pode ser executado somente depois da autenticação bem-sucedida no servidor com suporte a ADFS.
  
#### Soluções alternativas para a Vulnerabilidade de execução remota de código no ADFS - CVE-2009-2509
  
A Microsoft não identificou nenhuma solução alternativa para esta vulnerabilidade.
  
#### Perguntas frequentes sobre a Vulnerabilidade de execução remota de código no ADFS - CVE-2009-2509
  
**Qual é o escopo da vulnerabilidade?**    
Essa é uma vulnerabilidade de execução remota de código. Um invasor que explorar essa vulnerabilidade com êxito poderá executar qualquer ação no servidor IIS com os mesmos direitos de WPI (Identidade do Processo de Trabalho), que, por padrão, é configurado com privilégios da conta de Serviço de Rede.
  
**O que provoca a vulnerabilidade?**    
Esta vulnerabilidade existe porque o ADFS não valida corretamente cabeçalhos de solicitação enviados por clientes autenticados.
  
**Para que um invasor pode usar a vulnerabilidade?**    
Um invasor que explorar essa vulnerabilidade com êxito poderá executar qualquer ação no servidor IIS com os mesmos direitos de WPI (Identidade do Processo de Trabalho). O WPI é configurado com privilégios de conta de Serviço de rede por padrão. Os servidores IIS com páginas ASP cujo pool de aplicativos está configurado com um WPI que usa uma conta com privilégios administrativos podem ter mais impacto do que aqueles cujo pool de aplicativos está configurado com as configurações padrão de WPI.
  
**De que forma o invasor pode explorar a vulnerabilidade?**    
Um invasor pode tentar explorar a vulnerabilidade enviando uma solicitação HTTP especialmente criada a um servidor Web com ADFS habilitado. O invasor precisaria ser um usuário autenticado para explorar esta vulnerabilidade.
  
**Quais são os sistemas que mais correm riscos com a vulnerabilidade?**    
Os servidores Web com ADFS habilitado correm risco. Sistemas cliente não são afetados por esta vulnerabilidade.
  
**O que a atualização faz?**    
Esta atualização corrige a validação que servidores Web com ADFS habilitado aplicam aos cabeçalhos de solicitação enviados por um cliente da Web.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre essa vulnerabilidade por meio de divulgação responsável.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**

**Verificando a aplicação da atualização**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção **Orientação e ferramentas de detecção e implantação** neste boletim para obter mais informações.

-   **Verificação de Versão do Arquivo**

    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.

    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.
        **Observação** Dependendo da edição do sistema operacional, ou dos programas que são instalados em seu sistema, alguns arquivos que estão listados na tabela de informações de arquivo podem não estar instalados.
    5.  Na guia **Versão**, determine a versão do arquivo instalado no sistema, comparando-o com a versão documentada na tabela de informações de arquivos correta.
        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.

-   **Verificação da Chave do Registro**

    Você também pode ser capaz de verificar os arquivos que esta atualização de segurança instalou examinando as chaves do Registro na **Tabela de referência** nesta seção.

    Essas chaves do Registro podem não conter uma lista completa dos arquivos instalados. Além disso, talvez essas chaves do Registro não sejam criadas corretamente se um administrador ou um OEM integrar ou adicionar a atualização de segurança aos arquivos fonte da instalação do Windows.

#### Informações de implantação
  
**Instalando a atualização**
  
Quando você instala esta atualização de segurança, o instalador verifica se alguns dos arquivos que estão sendo atualizados no sistema foram anteriormente atualizados por um hotfix da Microsoft.
  
Para obter mais informações sobre a terminologia que aparece neste boletim, como *hotfix*, consulte o [Artigo 824684 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/824684).
  
Esta atualização de segurança oferece suporte às seguintes opções de instalação:
  
| Opção             | Descrição                                                                                                                                               |  
|-------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Exibe ajuda sobre as opções compatíveis.                                                                                                                |  
| **/quiet**        | Suprime a exibição de status ou mensagens de erro.                                                                                                      |  
| **/norestart**    | Quando combinado com **/quiet**, o sistema não é reiniciado após a instalação, mesmo que uma reinicialização seja necessária à conclusão da instalação. |
  
**Observação** Para obter mais informações sobre o instalador wusa.exe, consulte o [Artigo 934307 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/934307).
  
**Verificando a aplicação da atualização**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção **Orientação e ferramentas de detecção e implantação** neste boletim para obter mais informações.
  
-   **Verificação de Versão do Arquivo**
  
    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.
  
    1.  Clique em **Iniciar** e insira um nome de arquivo de atualização em **Iniciar Pesquisa**.  
    2.  Quando o arquivo for exibido em **Programas**, clique com o botão direito do mouse no nome do arquivo e clique em **Propriedades**.  
    3.  Na guia **Geral**, compare o tamanho do arquivo com as tabelas de informações sobre arquivos fornecidas no artigo da KB do boletim.  
    4.  Você também pode clicar na guia **Detalhes** e comparar as informações, como versão do arquivo e data da última modificação, com as tabelas de informações de arquivos fornecidas no artigo da KB do boletim.  
    5.  Finalmente, você também pode clicar na guia **Versões Anteriores** e comparar as informações de arquivo da versão anterior do arquivo com as informações sobre a versão nova ou atualizada.
  
### Outras informações
  
#### Microsoft Active Protections Program (MAPP)
  
Para melhorar as proteções de segurança para os clientes, a Microsoft fornece informações sobre vulnerabilidades aos principais fornecedores de software de segurança antes do lançamento de cada atualização de segurança mensal. Assim, os fornecedores de software de segurança podem usar essas informações sobre vulnerabilidades para fornecer proteções atualizadas aos clientes por meio de seus softwares ou dispositivos de segurança, como antivírus, sistemas de detecção de invasões baseados em rede ou sistemas de prevenção de invasões baseados em host. Para determinar se os fornecedores de software de segurança estão disponibilizando proteções ativas, visite os sites de proteções ativas fornecidos pelos parceiros do programa, listados em [Parceiros do MAPP (Microsoft Active Protections Program)](http://www.microsoft.com/security/msrc/mapp/partners.mspx).
