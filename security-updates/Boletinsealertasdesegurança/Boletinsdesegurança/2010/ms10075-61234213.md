Security Bulletin

Microsoft Security Bulletin MS10-075 - Crítica
==============================================

Vulnerabilidade no Serviço de Compartilhamento de Rede do Media Player pode permitir a execução remota de código (2281679)
--------------------------------------------------------------------------------------------------------------------------

Publicado: terça-feira, 12 de outubro de 2010

**Versão:** 1.0

### Informações Gerais

#### Sinopse

Esta atualização de segurança elimina uma vulnerabilidade relatada em particular no Serviço de Compartilhamento de Rede do Windows Media Player. A vulnerabilidade pode permitir a execução remota de código se um invasor enviar um pacote RTSP especialmente criado para um sistema afetado. No entanto, o acesso pela Internet à mídia caseira fica desabilitado, por padrão. Nesta configuração padrão, a vulnerabilidade pode ser explorada somente por um invasor que esteja na mesma sub-rede.

Esta atualização de segurança é classificada como Crítica para edições com suporte do Windows 7 e como Importante para todas as edições com suporte do Windows Vista. Para obter mais informações, consulte a subseção **Softwares afetados e não afetados**, nesta seção.

A atualização de segurança elimina a vulnerabilidade corrigindo a maneira como o Serviço de Compartilhamento de Rede do Windows Media Player manipula pacotes RTSP especialmente criados. Para obter mais informações sobre a vulnerabilidade, consulte a subseção Perguntas frequentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A maioria dos clientes tem o recurso de atualizações automáticas habilitado e não precisará tomar nenhuma providência porque esta atualização de segurança será baixada e instalada automaticamente. Os clientes que não têm o recurso de atualizações automáticas habilitado precisarão verificar as atualizações e instalar esta atualização manualmente. Para obter informações sobre opções de configuração específicas em atualizações automáticas, consulte o [Artigo 294871 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Para instalações feitas por administradores ou empresas, ou por usuários finais que desejam instalar esta atualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a atualização imediatamente usando software de gerenciamento de atualização ou verificando se há atualizações no serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a seção **Orientação e ferramentas de detecção e implantação** mais adiante neste boletim.

**Problemas conhecidos.** Nenhuma

Vulnerabilidade de uso após liberação de RTSP - CVE-2010-3225  
-------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade no Serviço de Compartilhamento de Rede do Windows Media Player que pode permitir a um usuário remoto enviar um pacote de rede especialmente criado a uma instância do serviço de fluxo de rede do aplicativo no contexto do aplicativo atual.
  
Para exibir esta vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2010-3225](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3225).
  
#### Fatores atenuantes da Vulnerabilidade de uso após liberação de RTSP - CVE-2010-3225
  
A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:
  
-   Por padrão, o acesso pela Internet a mídia caseira fica desabilitado. Nesta configuração padrão, a vulnerabilidade pode ser explorada somente por computadores localizados na mesma sub-rede.  
-   Em edições com suporte do Windows Vista, Windows 7 Professional, Windows 7 Enterprise e Windows 7 Ultimate, o Serviço de Compartilhamento de Rede do Windows Media Player não é habilitado, por padrão.
  
#### Soluções alternativas para a Vulnerabilidade de uso após liberação de RTSP - CVE-2010-3225
  
A solução alternativa refere-se a uma configuração ou alteração de configuração que não corrige a vulnerabilidade subjacente mas que ajudaria a bloquear vetores de ataque conhecidos antes de aplicar a atualização. A Microsoft testou as soluções alternativas e estados a seguir como parte do cenário que implica se uma solução alternativa reduziria funcionalidade:
  
-   **Desabilitar o Serviço de Compartilhamento de Rede do Windows Media Player em sistemas afetados**
  
    **Método interativo**
  
    **Observação** O uso incorreto do Editor do Registro pode causar problemas sérios e exigir a reinstalação do sistema operacional. A Microsoft não garante que os problemas resultantes do uso incorreto do Editor do Registro possam ser solucionados. Você é responsável pelo uso do Editor do Registro. Para obter informações sobre como editar o Registro, consulte o tópico da Ajuda "Alterar chaves e valores" do Editor do Registro (Regedit.exe) ou exiba os tópicos da Ajuda "Adicionar e excluir informações no Registro" e "Editar informações do Registro" no Regedt32.exe.
  
    1.  Clique em **Iniciar**, **Executar**, digite **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Localize e selecione a seguinte chave do Registro:  
        HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\services\\WMPNetworkSvc  
    3.  Clique no menu **Arquivo** e, em seguida, clique em **Exportar**.  
    4.  Na caixa de diálogo **Exportar arquivo do Registro**, digite **WMPNetworkSvc\_Backup.reg** e, em seguida, clique em **Salvar**.  
        **Observação** Isso criará um backup desta chave do Registro na pasta Meus Documentos por padrão.  
    5.  Selecione o valor **Iniciar** no painel direito do Editor do Registro. Pressione **Enter** para editar o valor da chave. Mude o campo de dados **Valor** para **4** e pressione **Enter**.  
    6.  Em um prompt de comando elevado, pare o serviço inserindo o seguinte comando:  
        `sc stop WMPNetworkSvc`
  
    **Script para implantação gerenciada**
  
    1.  Faça uma cópia de backup da chave de Registro, usando o seguinte comando:  
        `regedit /e WMPNetworkSvc_Backup.reg HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\WMPNetworkSvc`  
    2.  Salve o seguinte em um arquivo com extensão .REG (por exemplo: Disable\_WMPNetworkSvc.reg):  
        Editor do Registro do Windows Versão 5.00  
        HKEY\_LOCAL\_MACHINE\\System\\CurrentControlSet\\Services\\service\_name\]  
        Start"=dword:00000004  
    3.  Execute o script de Registro criado na etapa 2 na máquina de destino com o seguinte comando:  
        `Regedit /s Disable_WMPNetworkSvc.reg`  
    4.  Em um prompt de comando elevado, pare o serviço inserindo o seguinte comando:  
        `sc stop WMPNetworkSvc`
  
    **Impacto da solução alternativa.** O Compartilhamento de Mídia do Windows será desabilitado.
  
    **Como desfazer a solução alternativa.**
  
    Usando o método interativo:
  
    1.  Clique em **Iniciar**, **Executar**, digite **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Clique no menu **Arquivo** e, em seguida, clique em **Importar**.  
    3.  Na caixa de diálogo **Exportar Arquivo de Registro**, selecione o arquivo **WMPNetworkSvc\_Backup.reg** que foi criado no "Método interativo" acima e, em seguida, clique em **Abrir**.  
    4.  Saia do Editor do Registro e reinicie o computador.
  
    Usando o script de implantação gerenciada:
  
    1.  Execute o script de Registro criado na etapa 2 na máquina de destino com o seguinte comando:  
        `Regedit /s WMPNetworkSvc_Backup.reg`  
    2.  Reinicie o computador.
  
#### Perguntas frequentes sobre a Vulnerabilidade de uso após liberação de RTSP - CVE-2010-3225
  
**Qual é o escopo da vulnerabilidade?**    
Essa é uma vulnerabilidade de execução remota de código. Um invasor que explore com êxito esta vulnerabilidade poderá executar código no contexto de segurança da conta do Serviço de Rede.
  
**O que provoca a vulnerabilidade?**    
A vulnerabilidade é causada pela maneira como o Serviço de Compartilhamento de Rede do Windows Media Player processa pacotes RTSP.
  
**De que forma o invasor pode explorar a vulnerabilidade?**    
Um invasor pode explorar a vulnerabilidade criando e enviando um pacote RTSP especialmente criado a um sistema afetado.
  
**O que é RTSP?**    
O RTSP (Real Time Streaming Protocol) é um protocolo de nível de aplicativo para controlar a entrega de dados com propriedades de tempo real. O RTSP fornece uma estrutura extensível para possibilitar a entrega controlada sob demanda de dados em tempo real, como áudio e vídeo. As fontes de dados podem incluir tanto feeds de dados ao vivo quanto clipes armazenados. Este protocolo se destina a controlar várias sessões de entrega de dados, fornecer um meio de escolher canais de entrega, como UDP, multicast UDP e TCP, e proporcionar um meio de selecionar mecanismos de entrega baseados em RTP (RFC 1889). O RTSP está definido em [RFC 2326](http://www.ietf.org/rfc/rfc2326.txt).
  
O RTSP oferece suporte a ações de controle de reprodução, como parar, pausar, retroceder e avançar arquivos do Windows Media indexados. Você pode usar o RTSP para transmitir conteúdo para computadores que executem o Windows Media Player 9 Series ou posterior ou o Windows Media Services 9 Series ou posterior.
  
**O que é a conta Serviço de Rede?**    
A conta Serviço de Rede é uma conta incorporada que tem mais acesso a recursos e objetos do que os membros do grupo Usuários. Os serviços executados com a conta Serviço de Rede acessam recursos de rede usando as credenciais da conta do computador. O nome real da conta é AUTORIDADE NT\\SERVIÇO DE REDE. Para obter mais informações, consulte o artigo do MSDN, NetworkService Account.
  
**O que é o Serviço de Compartilhamento de Rede do Windows Media Player?**    
O Windows Media Player permite que você compartilhe arquivos em sua biblioteca com outros usuários em sua rede doméstica e navegue por arquivos que estão em bibliotecas que outros usuários em sua rede compartilham. O Compartilhamento de Mídia do Windows usa o Serviço de Compartilhamento de Rede do Windows Media Player.
  
**Como posso determinar se o Serviço de Compartilhamento de Rede do Windows Media Player está sendo executado em meu sistema?**    
Clique em **Iniciar**, **Executar**, digite **Services.msc** e clique em **OK**. No painel de controle Serviços, consulte se o status do Serviço de Compartilhamento de Rede do Windows Media Player é "Iniciado".
  
**Para que um invasor pode usar a vulnerabilidade?**    
O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle do sistema afetado no contexto de segurança da conta Serviço de Rede.
  
**Quais são os principais sistemas que correm riscos com a vulnerabilidade?**    
Sistemas que executam o Serviço de Compartilhamento de Rede do Windows Media Player (wmpnetwk.exe) para fazer o fluxo de mídia são os que mais correm risco.
  
**O que a atualização faz?**    
A atualização elimina a vulnerabilidade corrigindo a maneira como o Serviço de Compartilhamento de Rede do Windows Media Player manipula pacotes RTSP especialmente criados.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade por meio da divulgação coordenada de vulnerabilidades.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**    
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes quando este boletim de segurança foi lançado pela primeira vez.

**Avaliador de compatibilidade com atualizações e Kit de ferramentas de compatibilidade de aplicativos**
  
As atualizações frequentemente gravam nos mesmos arquivos e configurações do Registro necessários à execução dos aplicativos. É possível usar os componentes do [Avaliador de compatibilidade com atualizações](http://technet2.microsoft.com/windowsvista/en/library/4279e239-37a4-44aa-aec5-4e70fe39f9de1033.mspx?mfr=true) incluídos no [Kit de ferramentas de compatibilidade de aplicativos](http://www.microsoft.com/downloads/details.aspx?familyid=24da89e9-b581-47b0-b45e-492dd6da2971&displaylang=en) para agilizar o teste e a validação de atualizações do Windows com relação aos aplicativos instalados.
  
O Application Compatibility Toolkit (ACT) contém as ferramentas e a documentação necessárias para avaliar e atenuar problemas de compatibilidade com aplicativos antes da implantação do Microsoft Windows Vista, de uma atualização do Windows, de uma atualização de segurança da Microsoft ou de uma nova versão do Windows Internet Explorer em seu ambiente.
