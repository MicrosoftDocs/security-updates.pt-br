Boletim de Segurança da Microsoft MS16-007 - Importante
=======================================================

Atualização de segurança para o Microsoft Windows para corrigir execução remota de código (3124901) 
----------------------------------------------------------------------------------------------------

Publicado em: 12.01.16

**Versão:** 1.0

Sinopse
-------

<span id="sectionToggle0"></span>
Esta atualização de segurança resolve vulnerabilidades no Microsoft Windows. A mais grave destas vulnerabilidades poderá permitir a execução remota de código se um atacante conseguir iniciar sessão em um sistema de destino e executar um aplicativo especialmente criado.

Essa atualização de segurança foi classificada como Importante para todas as versões com suporte do Microsoft Windows. Para obter mais informações, consulte a seção **Softwares Afetados**.

A atualização de segurança aborda vulnerabilidades ao:

-   Corrigir a forma como o Windows valida entradas antes de carregar arquivos DLL,
-   Corrigir a forma como o Microsoft DirectShow valida entradas de usuários
-   Aplicar as configurações padrão de não permitir o início de sessão para contas sem senhas.

Para obter mais informações sobre as vulnerabilidades, consulte a seção **Informações sobre vulnerabilidade**.

<span id="KBArticle"></span>
Para obter mais informações sobre essa atualização, consulte o [artigo 3124901 da Base de Dados de Conhecimento da Microsoft](https://support.microsoft.com/pt-br/kb/3124901).

Softwares Afetados e Classificações de Gravidade da Vulnerabilidade
-------------------------------------------------------------------

<span id="sectionToggle1"></span>
As seguinte versões ou edições de software foram afetadas. As versões ou edições que não estão listadas já passaram de seu ciclo de vida do suporte ou não foram afetadas. Para determinar o ciclo de vida do suporte para sua versão ou edição de software, visite o site [Ciclo de vida do suporte da Microsoft](http://support.microsoft.com/default.aspx?scid=fh;%5Bln%5D;lifecycle). 

As classificações de gravidade indicadas para cada software afetados assumem o potencial máximo do impacto da vulnerabilidade. Para obter informações referentes à probabilidade de exploração da vulnerabilidade em relação à sua classificação de gravidade e ao impacto à segurança, em até 30 dias a partir do lançamento deste boletim de segurança, consulte o Índice de exploração no [Resumo de boletins de janeiro](https://technet.microsoft.com/pt-br/library/security/ms16-jan).

Múltiplas vulnerabilidades de elevação de privilégio de carregamento de DLL
---------------------------------------------------------------------------

Múltiplas vulnerabilidades de elevação de privilégios existem quando o Windows valida incorretamente entradas antes de carregar arquivos de biblioteca de vínculo dinâmico (DDL). Um atacante que tenha explorado com êxito estas vulnerabilidades pode elevar seus privilégios em um sistema de destino.

Para explorar as vulnerabilidades, o atacante precisa primeiro fazer logon no sistema destino. Em seguida, ele poderia executar um aplicativo especialmente criado capaz de explorar as vulnerabilidades e assumir o controle do sistema afetado. A atualização elimina as vulnerabilidades, corrigindo a forma como o Windows valida as entradas antes de carregar arquivos DLL.

Vulnerabilidade de execução remota de código de corrupção de heap do DirectShow - CVE-2016-0015  
-----------------------------------------------------------------------------------------------
  
Existe uma vulnerabilidade de execução remota de código quando O [Microsoft DirectShow](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) valida incorretamente entradas de usuários. O atacante que explorar essa vulnerabilidade com êxito poderá fazer com que um código arbitrário seja executado no contexto do usuário atual. Se um usuário estiver conectado com direitos administrativos, o atacante poderá assumir o controle total do sistema afetado. O atacante poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas estão configuradas para ter menos direitos de usuário podem ser menos afetados do que os usuários que têm direitos de administrador.
  
Para um ataque ser bem-sucedido, essa vulnerabilidade requer que um usuário abra um arquivo especialmente criado. Em um cenário de ataque por e-mail, o atacante pode explorar a vulnerabilidade enviando um link especialmente criado para o usuário e persuadindo-o a abri-lo. A atualização de segurança aborda a vulnerabilidade modificando a forma como o DirectShow valida entradas de usuários.
  
A Microsoft recebeu informações sobre esta vulnerabilidade por meio da divulgação coordenada de vulnerabilidades. Quando esse boletim de segurança foi originalmente emitido, a Microsoft ainda não tinha recebido informações que indicassem que a vulnerabilidade tinha sido usada de forma pública para atacar clientes.
  
### Fatores atenuantes

Múltiplas vulnerabilidades de execução remota de código de carregamento de DLL  
------------------------------------------------------------------------------
  
Múltiplas vulnerabilidades de execução remota de código existem quando o Windows valida incorretamente entradas antes de carregar arquivos de biblioteca de vínculo dinâmico (DDL). O atacante que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado. O atacante poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas estão configuradas para ter menos direitos de usuário podem ser menos afetados do que os usuários que têm direitos de administrador.
  
Para explorar as vulnerabilidades, primeiramente, o atacante precisa fazer logon no sistema de destino e, em seguida, executar um aplicativo especialmente criado. As atualizações eliminam as vulnerabilidades, corrigindo a forma como o Windows valida as entradas antes de carregar arquivos DLL.

Vulnerabilidade de desvio de segurança do protocolo da área de trabalho remota do Windows - CVE-2016-0019  
---------------------------------------------------------------------------------------------------------
  
A vulnerabilidade de desvio de recurso de segurança existe no protocolo Windows Remote Desktop Protocol (RDP) que é causada quando anfitriões do Windows 10 que executam serviços RDP não conseguem impedir o logon remoto em contas sem senhas definidas. O atacante que explorar com êxito esta vulnerabilidade poderá obter acesso ao anfitrião remoto como outro usuário, possivelmente com privilégios elevados.
  
O atacante pode explorar esta vulnerabilidade usando uma versão anterior do cliente RDP para se conectar ao anfitrião do Windows 10. Uma vez conectado, o atacante pode gerar uma lista de contas de usuários no anfitrião e tentar iniciar a sessão como esses usuários. Se uma das contas de usuários não tiver uma senha configurada, o atacante pode iniciar sessão como esse usuário, apesar de existir uma configuração do sistema predefinida que limita o acesso a contas sem senha apenas a um início de sessão local. A atualização de segurança elimina a vulnerabilidade aplicando as configurações padrão de não permitir o início de sessão para contas sem senhas.
  
A Microsoft recebeu informações sobre esta vulnerabilidade por meio da divulgação coordenada de vulnerabilidades. No momento da publicação original deste boletim, a Microsoft não estava ciente de nenhum ataque tentando explorar essa vulnerabilidade.
