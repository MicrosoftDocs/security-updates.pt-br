Vulnerabilidade no Microsoft Internet Information Services (IIS) usando o Active Server Pages pode permitir a execução remota de código (917537)
------------------------------------------------------------------------------------------------------------------------------------------------

Publicado: terça-feira, 11 de julho de 2006 | Atualizado: quarta-feira, 12 de julho de 2006

**Sinopse:**

Esta atualização elimina uma vulnerabilidade descoberta recentemente e reportada em particular. A vulnerabilidade é documentada neste boletim na seção “Detalhes da vulnerabilidade”.

O invasor deve ter credenciais de logon válidas, mas se um servidor tiver sido intencionalmente configurado de modo a permitir que usuários, anônimos ou autenticados, carreguem conteúdo da Web como páginas .ASP para sites, o servidor pode ser explorado por esta vulnerabilidade.

Detalhes da Vulnerabilidade
---------------------------

<span></span>
#### Vulnerabilidade do Internet Information Services (IIS) usando Active Server Pages malformadas - CVE-2006-0026

Há uma vulnerabilidade que permite a execução remota de código no Internet Information Services (IIS). Um invasor pode explorar a vulnerabilidade criando um arquivo especial do Active Server Pages (ASP), que pode permitir a execução remota de código se o Internet Information Services (IIS) processar o arquivo especialmente criado. O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado.

#### Fatores atenuantes da vulnerabilidade do Internet Information Services (IIS) usando Active Server Pages (ASP) malformadas - CVE-2006-0026:

-   No IIS 5.0 e no IIS 5.1, aplicativos ativados para ASP são executados, por padrão, no aplicativo 'Agrupado do Processo', o que significa que são executados no DLLHOST.exe, executado no contexto da conta IWAM\_&lt;nomedamáquina&gt; com baixos privilégios.
-   Por padrão, o IIS 5.1 no Windows XP Professional e o IIS 6.0 no Windows Server 2003 não são ativados.
-   Por padrão, o ASP não é ativado no IIS 6.0. Se o ASP for ativado, será executado no contexto de um processo de trabalho W3WP.exe executado como a conta 'NetworkService' com baixos privilégios.
-   Um invasor deve ter credenciais de logon válidas para explorar esta vulnerabilidade. Entretanto, se um servidor tiver sido intencionalmente configurado de modo a permitir que usuários, anônimos ou autenticados, carreguem conteúdo da Web como páginas .ASP para sites, o servidor pode ser explorado por esta vulnerabilidade.

#### Soluções alternativas para a vulnerabilidade do Internet Information Services (IIS) usando Active Server Pages malformadas - CVE-2006-0026

Não identificamos nenhuma solução alternativa para essa vulnerabilidade.

#### Perguntas freqüentes sobre a vulnerabilidade do Internet Information Services (IIS) usando Active Server Pages malformadas - CVE-2006-0026:

**Qual é o escopo da vulnerabilidade?**  
Há uma vulnerabilidade que permite a execução remota de código no Internet Information Services (IIS), devido à maneira como o ISS lida com Active Server Pages (ASP). Um invasor pode explorar a vulnerabilidade criando um arquivo especial do Active Server Pages (ASP), que pode permitir a execução remota de código se processado pelo Internet Information Services (IIS). O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado.

**O que provoca a vulnerabilidade?**  
Um buffer não verificado no IIS.

**O que é Active Server Pages (ASP)?**  
Microsoft Active Server Pages (ASP) é uma tecnologia de script do lado do servidor que pode ser usada para criar aplicativos da Web dinâmicos e interativos. Uma página ASP é uma página HTML que contém scripts de servidor que são processados pelo servidor Web antes de serem enviados ao navegador do usuário.

**Para que um invasor pode usar a vulnerabilidade?**  
Um invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total do sistema afetado.

**Quem pode explorar a vulnerabilidade?**  
Um invasor precisaria ter credenciais de logon válidas no servidor para explorar a vulnerabilidade. Entretanto, se um servidor tiver sido intencionalmente configurado de modo a permitir que usuários, anônimos ou autenticados, carreguem conteúdo da Web como páginas .ASP para sites, o servidor pode ser explorado por esta vulnerabilidade.

**De que forma o invasor pode explorar a vulnerabilidade?**  
Um invasor pode tentar explorar a vulnerabilidade criando um arquivo ASP especial e carregando-o em um sistema afetado. Se o IIS processar o arquivo, este poderá fazer com que o sistema afetado execute códigos.

**Quais são os principais sistemas que correm riscos com a vulnerabilidade?**  
Sistemas baseados no Windows 2000 Service Pack 4 que executam o IIS 5.0 estão correndo risco relativo a essa vulnerabilidade, visto que o IIS é ativado por padrão. O Windows XP Professional e o Windows Server 2003 também estarão em risco se o serviço for ativado.

**A vulnerabilidade pode ser explorada pela Internet?**  
Sim. Um invasor pode tentar explorar esta vulnerabilidade pela Internet se obtiver acesso para gravar no servidor. As práticas recomendadas de firewall e as configurações de firewall padrão podem ajudar na proteção contra ataques originados na Internet. A Microsoft forneceu informações sobre como você pode ajudar a proteger o seu PC. Os usuários finais podem visitar o site [Proteja seu PC](http://www.microsoft.com/brasil/proteja). Os profissionais de TI podem visitar o site [Centro de Orientacoes de Seguranca](http://www.microsoft.com/brasil/security/guidance) (em inglês).

**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**  
Não. A Microsoft recebeu informações sobre essa vulnerabilidade por meio de divulgação responsável. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido divulgada publicamente quando este boletim de segurança foi lançado pela primeira vez.

**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**  
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes e não testemunhou nenhum exemplo de publicação de código de verificação de conceito, quando este boletim de segurança foi lançado pela primeira vez.

**Requisitos de reinicialização**

Essa atualização não requer reinicialização. O instalador interrompe os serviços necessários, aplica a atualização e reinicia-os em seguida. Contudo, se esses serviços necessários não puderem ser interrompidos por algum motivo, ou se os arquivos necessários estiverem em uso, a atualização exigirá uma reinicialização. Se isso ocorrer, uma mensagem que aconselha a reinicialização será exibida.

Para reduzir a possibilidade de uma reinicialização ser solicitada, interrompa todos os serviços afetados e feche todos os aplicativos que possam usar os arquivos afetados antes de instalar a atualização de segurança. **Requisitos de reinicialização**

Essa atualização não requer reinicialização. O instalador interrompe os serviços necessários, aplica a atualização e reinicia-os em seguida. Contudo, se esses serviços necessários não puderem ser interrompidos por algum motivo, ou se os arquivos necessários estiverem em uso, a atualização exigirá uma reinicialização. Se isso ocorrer, uma mensagem que aconselha a reinicialização será exibida.

Para reduzir a possibilidade de uma reinicialização ser solicitada, interrompa todos os serviços afetados e feche todos os aplicativos que possam usar os arquivos afetados antes de instalar a atualização de segurança. Para obter mais informações sobre os motivos pelos quais você será solicitado a reiniciar, consulte o [artigo 887012 (em inglês) da Base de Conhecimento Microsoft](http://support.microsoft.com/kb/887012).
