Security Bulletin

Microsoft Security Bulletin MS08-037 - Importante
=================================================

Vulnerabilidades no DNS podem permitir falsificação (953230)
------------------------------------------------------------

Publicado: terça-feira, 8 de julho de 2008 | Atualizado: quarta-feira, 9 de dezembro de 2009

**Versão:** 3.1

### Informações Gerais

#### Sinopse

Esta atualização de segurança resolve duas vulnerabilidades relatadas em particular no Sistema de Nomes de Domínio (DNS) do Windows, que podem permitir falsificação. Estas vulnerabilidades existem tanto no cliente DNS quanto no servidor DNS e podem permitir que um invasor remoto redirecione o tráfego de rede destinado a sistemas na Internet aos próprios sistemas do invasor.

Esta atualização de segurança é classificada como importante para todas as edições do Microsoft Windows 2000, Windows XP, Windows Vista, Windows Server 2003 e Windows Server 2008 com suporte. Para obter mais informações, consulte a subseção **Software afetado e não afetado** nesta seção.

A atualização de segurança soluciona as vulnerabilidades usando as IDs de transação do DNS fortemente aleatórias, usando soquetes aleatórios para consultas UDP e atualizando a lógica usada para gerenciar o cache DNS. Para obter mais informações sobre a vulnerabilidade, consulte a subseção Perguntas freqüentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A Microsoft recomenda que os clientes instalem a atualização o mais rápido possível.

**Problemas conhecidos.** O [Artigo 953230 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/953230) documenta os problemas atualmente conhecidos que os clientes podem enfrentar ao instalar esta atualização de segurança.

#### Software afetado e não afetado

O software a seguir foi testado para determinar quais versões ou edições foram afetadas. O ciclo de vida do suporte das outras versões ou edições já terminou ou elas não são afetadas. Para determinar o ciclo de vida do suporte para sua versão ou edição de software, visite o site [Ciclo de vida do Suporte Microsoft.](http://go.microsoft.com/fwlink/?linkid=21742)

Perguntas freqüentes relacionadas a esta atualização de segurança  
-----------------------------------------------------------------
  

**Por que este boletim foi relançado em 8 de dezembro de 2009?**       
A Microsoft relançou este boletim para oferecer novamente a atualização para o Cliente DNS no Microsoft Windows 2000 Service Pack 4 (KB951748), para fornecer IDs de transações de DNS fortemente aleatórias para um caminho de código adicional. Ao contrário de outras plataformas Windows, no Microsoft Windows 2000, há dois caminhos de código para transações de DNS. A atualização anterior só forneceu a randomização de ID da transação em um dos caminhos de código. O relançamento desta atualização fornece a mesma randomização de ID da transação para o outro caminho de código no Microsoft Windows 2000. Os clientes que instalaram previamente a atualização para o Cliente DNS no Microsoft Windows 2000 Service Pack 4 (KB951748) precisam instalar a atualização oferecida novamente de forma automática. Nenhuma outra atualização é afetada por este relançamento.
  
**Por que este boletim de segurança foi revisado em 13 de janeiro de 2009?**    
A Microsoft revisou este boletim para comunicar que a atualização oferecida por ele pode não ter sido oferecida corretamente para todos os sistemas que executam o Windows XP Service Pack 3. O?problema de detecção e implantação foi corrigido, e a atualização será oferecida corretamente aos clientes com sistemas Windows XP Service Pack 3 que ainda não aplicaram a atualização deste boletim. Não houve nenhuma alteração aos binários desta atualização. Os clientes que atualizaram seus sistemas com êxito não necessitam reinstalar esta atualização.
  
**Por que este boletim de segurança foi revisado em 25 de julho de 2008?**    
A Microsoft revisou este boletim de segurança para adicionar três novos problemas conhecidos à seção **Perguntas frequentes relacionadas a esta atualização de segurança**. Essas Perguntas frequentes são:
  
-   **Quais são os problemas de conflito de portas que os clientes podem enfrentar após instalar esta atualização?**    
    Os servidores DNS que executam serviços adicionais que requerem portas estáticas UDP no intervalo de portas alocadas pelo servidor DNS para seu pool de soquete podem enfrentar conflitos de porta que podem resultar em falhas do serviço. Para obter mais informações, leia o [Artigo 953230 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/953230), que documenta os problemas atualmente conhecidos que os clientes podem enfrentar ao instalar esta atualização de segurança.  
-   **Quais são os problemas de conflito de portas que os clientes do SBS podem enfrentar após instalar esta atualização?**    
    Edições com suporte do Windows Small Business Server (SBS) 2003 contêm o mesmo código afetado do Windows Server 2003 Service Pack 1 e do Windows Server 2003 Service Pack 2. No entanto, sistemas SBS podem enfrentar falhas de serviço ou podem não funcionar corretamente após a instalação da atualização de segurança para servidor DNS oferecida neste boletim. Para obter mais informações, leia o [Artigo 953230 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/953230), que documenta os problemas atualmente conhecidos que os clientes podem enfrentar ao instalar esta atualização de segurança.  
-   **Quais problemas os dispositivos de firewalls de perímetro e NAT de perímetro podem enfrentar após a instalação desta atualização?**    
    Após a instalação da atualização de segurança oferecida neste boletim em um computador baseado no Windows, as consultas de DNS do host através de um firewall podem não usar mais portas de origem aleatória. Para obter mais informações, leia o [Artigo 953230 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/953230), que documenta os problemas atualmente conhecidos que os clientes podem enfrentar ao instalar esta atualização de segurança.

**Quais são os problemas de desinstalação que os clientes podem enfrentar ao instalar estas atualizações de segurança?**    
Edições dos sistemas Microsoft Windows 2000 e Windows 2003 com suporte receberão a atualização do servidor DNS assim como as atualizações do cliente DNS. A atualização de servidor DNS e as atualizações de cliente DNS compartilham binários e devem ser desinstaladas na ordem inversa em que foram instaladas para evitar a regressão dos binários compartilhados às versões anteriores. O [Artigo 823836 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/823836) documenta os problemas atualmente conhecidos que os clientes podem enfrentar ao desinstalar estas atualizações de segurança.
  
**Quais são os problemas de conexão de soquete aleatória que os clientes podem enfrentar ao instalar esta atualização de segurança?**    
Por padrão, as atualizações de DNS oferecidas por este boletim de segurança tiram proveito do maior número de soquetes disponíveis para oferecer o máximo de entropia. No entanto, se o usuário definiu intervalos de portas no Registro, as atualizações respeitarão as configurações definidas pelo usuário e somente alocarão os soquetes definidos.
  
Os intervalos de soquetes podem ser definidos no seguinte local do Registro:  
**HKLM\\System\\CurrentControlSet\\Services\\DNS\\Parameters Reg key Name: SocketPoolSize**  
**Observação** O serviço DNS deve ser reiniciado para implementar estas alterações.
  
**O que a definição do intervalo de pool de soquetes faz?**    
Pode ser necessário definir o intervalo de soquetes do qual o DNS pode escolher para evitar conflitos com outros aplicativos ou serviços que necessitam do mesmo pool de soquetes para sua comunicação. Consulte [MaxUserPort](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/regentry/58791.mspx?mfr=true) e o [Artigo 812873 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/812873) para obter mais detalhes sobre estas configurações de chave do Registro.
  
**Quais são as diferenças entre sistemas operacionais ao definir os intervalos de pool de soquetes?**    
A chave do Registro MaxUserPort tem significado diferente no Windows Vista e Windows Server 2008 e no Microsoft Windows Server 2000 e Windows Server 2003. O [Artigo 929851 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/929851) detalha a alteração em comportamento no Windows Vista e Windows Server 2008.
  
Em Microsoft Windows Server 2000 e Windows Server 2003, a configuração do MaxUserPort define o ponto final do intervalo dinâmico de porta. O intervalo inicia em 1024 e continua até o valor definido pelo usuário na configuração da chave do Registro MaxUserPort. Depois que instalar as atualizações oferecidas por este boletim de segurança, o comportamento padrão no Microsoft Windows Server 2000 e no Windows Server 2003 será alocar soquetes aleatoriamente do intervalo de porta 49152 a 65535. Se o intervalo MaxUserPort for definido, então as portas serão alocadas aleatoriamente de 1024 até o valor definido na configuração da chave do Registro MaxUserPort. Consulte o [Artigo 812873 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/812873) para obter mais informações sobre como reservar intervalos de portas no Microsoft Windows 2000 Server e no Windows Server 2003.
  
Em Windows Vista e Windows Server 2008, a configuração de MaxUserPort define o ponto inicial do intervalo dinâmico de porta. Por padrão, o intervalo no Windows Vista e no Windows Server 2008 vai de 49152 a 65535.

Vulnerabilidade de entropia de soquete insuficiente do DNS - CVE-2008-1447  
--------------------------------------------------------------------------
  

Existe uma vulnerabilidade de falsificação no cliente DNS do Windows e no servidor DNS do Windows. Esta vulnerabilidade pode permitir que um invasor não autenticado remoto falsifique, de maneira rápida e confiável, respostas e insira registros no cache do cliente ou servidor DNS, redirecionando, assim, o tráfego de Internet.
  
Para exibir essa vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2008-1447](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1447).
  
#### Fatores atenuantes para a vulnerabilidade de entropia de soquete insuficiente no DNS - CVE-2008-1447
  
A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade.
  
-   Protocolos criptográficos que operam acima das camadas IP e TCP, como IPsec ou SSL/TLS, podem impedir que um invasor monitore ou interfira no tráfego redirecionado.
  
#### Soluções alternativas para a vulnerabilidade de entropia de soquete insuficiente no DNS - CVE-2008-1447
  
A solução alternativa refere-se a uma configuração ou alteração de configuração que não corrige a vulnerabilidade subjacente mas que ajudaria a bloquear vetores de ataque conhecidos antes de aplicar a atualização.
  
-   A Microsoft não identificou nenhuma solução alternativa para esta vulnerabilidade.
  
#### Perguntas frequentes sobre a vulnerabilidade de entropia de soquete insuficiente no DNS - CVE-2008-1447
  
**Qual é o escopo da vulnerabilidade?**    
Existe uma vulnerabilidade de falsificação no cliente DNS do Windows e no servidor DNS. Um invasor que explorar com êxito esta vulnerabilidade pode inserir endereços arbitrários no cache DNS.
  
**O que provoca a vulnerabilidade?**    
O serviço DNS do Windows no cliente do Windows e servidor DNS não fornece entropia suficiente ao executar consultas de DNS.
  
**O que é DNS (Domain Name System)?**    
O DNS (Domain Name System) é um dos conjuntos de protocolos padrão do setor que abrangem TCP/IP. O DNS é implementado usando-se dois componentes de software: o servidor de DNS e o cliente de DNS (ou resolvedor). Ambos os componentes são executados como aplicativos de serviços de segundo plano. Os recursos de rede são identificados por endereços IP numéricos, mas estes endereços IP são difíceis de lembrar para os usuários de rede. O banco de dados do DNS contém registros que traçam nomes alfanuméricos de fácil utilização para recursos de rede, como www.microsoft.com, ao endereço IP usado por esses recursos para comunicação. Desta maneira, o DNS age como um dispositivo mnemônico, tornando mais fácil para os usuários da rede lembrar os recursos da rede. Para obter mais informações e ver diagramas lógicos que ilustram como o DNS ajusta-se a outras tecnologias do Windows, leia o artigo [O que é DNS](http://technet2.microsoft.com/windowsserver/en/library/ff937311-03ce-4d04-b72c-b39c4d51cb361033.mspx) (em inglês).
  
**O que é Cache DNS?**    
O serviço de resolução de cache DNS (Sistema de Nomes de Domínio) é um serviço que salva as respostas para consultas DNS de modo que o servidor DNS não seja repetidamente consultado para as mesmas informações. Para obter mais informações, consulta o artigo da Technet [DNSCache](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/regentry/30643.mspx?mfr=true) (em inglês) ou o artigo da Technet [DNS Resolver Cache Service](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/cnet/cnbc_imp_qxht.mspx?mfr=true) (em inglês). Consulte o artigo da Technet [Attack detection](http://www.microsoft.com/technet/isa/2004/help/fw_alertattack.mspx?mfr=true) (em inglês) para obter mais informações sobre DNS Cache Poisoning.
  
**Para que um invasor pode usar a vulnerabilidade?**    
Um invasor que explorar com êxito esta vulnerabilidade poderá inserir endereços arbitrários no cache DNS, também conhecido como DNS Cache Poisoning.
  
**De que forma o invasor pode explorar a vulnerabilidade? **  
Um invasor pode enviar consultas específicas a um cliente ou servidor DNS vulnerável e, ao mesmo tempo, responder novamente de uma maneira que permita que o invasor insira dados do DNS falsos ou fraudulentos. O invasor pode redirecionar tráfego de Internet de locais legítimos a um endereço da escolha do invasor.
  
**Quais são os sistemas que mais correm riscos com a vulnerabilidade?**    
Qualquer sistema Windows conectado a uma rede ou Internet corre risco. O Windows Vista e Windows Server 2008 não são afetados por esta vulnerabilidade.
  
**O que a atualização faz? **  
A atualização elimina esta vulnerabilidade usando valores de ID de transação de DNS fortemente aleatórios e soquetes UDP aleatórios para consultas remotas.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre essa vulnerabilidade por meio de divulgação responsável.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**    
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes e não testemunhou nenhum exemplo de publicação de código de verificação de conceito, quando este boletim de segurança foi lançado pela primeira vez.
  
Vulnerabilidade no DNS Cache Poisoning - CVE-2008-1454  
------------------------------------------------------
  

Existe uma vulnerabilidade no envenenamento de cache no Servidor DNS do Windows. A vulnerabilidade pode permitir que um invasor remoto não autenticado envie respostas especialmente criadas a solicitações de DNS feitas por sistemas vulneráveis, envenenando, assim, o cache DNS e redirecionando o tráfego de Internet de locais legítimos.
  
Para exibir essa vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2008-1454](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1454).

Consulte o artigo da Technet [Attack detection](http://www.microsoft.com/technet/isa/2004/help/fw_alertattack.mspx?mfr=true) (em inglês) para obter mais informações sobre DNS Cache Poisoning.
  
**Para que um invasor pode usar a vulnerabilidade?**

**Quais são os sistemas que mais correm riscos com a vulnerabilidade?**

A atualização elimina a vulnerabilidade, corrigindo o processamento interno do DNS para evitar envenenamento de cache.

**Instalando a atualização**

Quando você instala esta atualização de segurança, o instalador verifica se alguns dos arquivos que estão sendo atualizados no sistema foram anteriormente atualizados por um hotfix da Microsoft.

Se você já tiver instalado um hotfix para atualizar um desses arquivos, o instalador copiará os arquivos RTMQFE, SP1QFE ou SP2QFE para o sistema. Do contrário, o instalador copiará os arquivos RTMGDR, SP1GDR ou SP2GDR para o sistema. As atualizações de segurança podem não conter todas as variações desses arquivos. Para obter mais informações sobre esse comportamento, consulte o [Artigo 824994 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/824994).

**Verificando a aplicação da atualização**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção **Orientação e ferramentas de detecção e implantação** neste boletim para obter mais informações.

-   **Verificação de Versão do Arquivo**

    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.

    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.
        **Observação** Dependendo da edição do sistema operacional, ou dos programas que são instalados em seu sistema, alguns arquivos que estão listados na tabela de informações de arquivo podem não estar instalados.
    5.  Na guia **Versão**, determine a versão do arquivo instalado no sistema, comparando-o com a versão documentada na tabela de informações de arquivos correta.
        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.

-   **Verificação da Chave do Registro**

    Você também pode ser capaz de verificar os arquivos que esta atualização de segurança instalou examinando as chaves do Registro na **Tabela de referência** nesta seção.

    Essas chaves do Registro podem não conter uma lista completa dos arquivos instalados. Além disso, talvez essas chaves do Registro não sejam criadas corretamente se um administrador ou um OEM integrar ou adicionar a atualização de segurança aos arquivos fonte da instalação do Windows.

-   **Verificação de Versão do Arquivo**

    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.

    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.
        **Observação** Dependendo da edição do sistema operacional, ou dos programas que são instalados em seu sistema, alguns arquivos que estão listados na tabela de informações de arquivo podem não estar instalados.
    5.  Na guia **Versão**, determine a versão do arquivo instalado no sistema, comparando-o com a versão documentada na tabela de informações de arquivos correta.
        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.

#### Informações de implantação
  
**Instalando a atualização**
  
Quando você instala esta atualização de segurança, o instalador verifica se alguns dos arquivos que estão sendo atualizados no sistema foram anteriormente atualizados por um hotfix da Microsoft.
  
Para obter mais informações sobre a terminologia que aparece neste boletim, como *hotfix*, consulte o [Artigo 824684 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/824684).
  
Esta atualização de segurança oferece suporte às seguintes opções de instalação:
  
Opções de instalação da atualização de segurança para as quais há suporte | Opção             | Descrição                                                                                                                                               |  
|-------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Exibe ajuda sobre as opções compatíveis.                                                                                                                |  
| **/quiet**        | Suprime a exibição de status ou mensagens de erro.                                                                                                      |  
| **/norestart**    | Quando combinado com **/quiet**, o sistema não é reiniciado após a instalação, mesmo que uma reinicialização seja necessária à conclusão da instalação. |
  
**Observação** Para obter mais informações sobre o instalador wusa.exe, consulte o [Artigo 934307 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/934307).
  
**Verificando a aplicação da atualização**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção **Orientação e ferramentas de detecção e implantação** neste boletim para obter mais informações.
  
-   **Verificação de Versão do Arquivo**
  
    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.
  
    1.  Clique em **Iniciar** e insira um nome de arquivo de atualização em **Iniciar Pesquisa**.  
    2.  Quando o arquivo for exibido em **Programas**, clique com o botão direito do mouse no nome do arquivo e clique em **Propriedades**.  
    3.  Na guia **Geral**, compare o tamanho do arquivo com as tabelas de informações sobre arquivos fornecidas no artigo da KB do boletim.  
    4.  Você também pode clicar na guia **Detalhes** e comparar as informações, como versão do arquivo e data da última modificação, com as tabelas de informações de arquivos fornecidas no artigo da KB do boletim.  
    5.  Finalmente, você também pode clicar na guia **Versões Anteriores** e comparar as informações de arquivo da versão anterior do arquivo com as informações sobre a versão nova ou atualizada.
