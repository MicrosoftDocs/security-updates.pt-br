Security Bulletin

Microsoft Security Bulletin MS08-055 - Crítica
==============================================

Vulnerabilidade no Microsoft Office pode permitir a execução remota de código (955047)
--------------------------------------------------------------------------------------

Publicado: terça-feira, 9 de setembro de 2008 | Atualizado: quarta-feira, 10 de setembro de 2008

**Versão:** 1.1

### Informações Gerais

#### Sinopse

Esta atualização de segurança elimina uma vulnerabilidade relatada em particular no Microsoft Office. A vulnerabilidade pode permitir a execução remota de código se um usuário clicar em um URL do OneNote especialmente criado. O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.

Esta atualização de segurança é classificada como Crítica para as edições do OneNote 2007 com suporte e classificada como Importante para as edições do Microsoft Office XP, Microsoft Office 2003 e Microsoft Office System 2007 com suporte. Para obter mais informação, consulte a subseção **Software afetado e não afetado** nesta seção.

Esta atualização de segurança elimina a vulnerabilidade, modificando a maneira como o Microsoft Office valida os localizadores uniformes de recursos. Para obter mais informações sobre a vulnerabilidade, consulte a subseção Perguntas freqüentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A Microsoft recomenda que os clientes apliquem a atualização imediatamente.

**Problemas conhecidos.** Nenhuma

#### Software afetado e não afetado

O software a seguir foi testado para determinar quais versões ou edições foram afetadas. O ciclo de vida do suporte das outras versões ou edições já terminou ou elas não são afetadas. Para determinar o ciclo de vida do suporte para sua versão ou edição de software, visite o site [Ciclo de vida do Suporte Microsoft.](http://go.microsoft.com/fwlink/?linkid=21742)

Perguntas freqüentes relacionadas a esta atualização de segurança
-----------------------------------------------------------------


**Por que este Boletim foi revisado em 10 de setembro de 2008?**  
A Microsoft revisou este boletim para corrigir as opções de instalação e informações sobre implantação para o OneNote 2007 e adicionar o Microsoft Office PowerPoint Viewer 2003, Microsoft Office Word Viewer 2003, Microsoft Office Word Viewer 2003 Service Pack 3, Microsoft Office Excel Viewer 2003, Microsoft Office Excel Viewer 2003 Service Pack 3, Microsoft Office Excel Viewer, Microsoft Office PowerPoint 2007 Viewer, Microsoft Office PowerPoint Viewer 2007 Service Pack 1, Pacote de compatibilidade do Microsoft Office para formatos de arquivo do Word, Excel e PowerPoint 2007 e Pacote de compatibilidade do Microsoft Office para formatos de arquivos do Word, Excel e PowerPoint 2007 Service Pack 1 à lista de softwares não afetados. A Microsoft também corrigiu as entradas na seção Perguntas freqüentes que explicam por que esta atualização é oferecida a sistemas com softwares não afetados.

**Onde estão os detalhes das informações sobre o arquivo?**  
Informações detalhadas sobre o arquivo podem ser encontradas no [Artigo 955047 (em inglês) da Base de Conhecimento Microsoft](http://support.microsoft.com/kb/955047).

**O boletim MS08-052 também descreve as vulnerabilidades no Microsoft Office XP Service Pack 3? Como o boletim MS08-052 está relacionado a este boletim (MS08-055)?**  
Como parte do modelo cumulativo de serviços para Microsoft Office XP, esta atualização de segurança para Microsoft Office XP Service Pack 3 (KB953405) também aborda as vulnerabilidades descritas no boletim MS08-052. Os usuários com Microsoft Office XP Service Pack 3 instalado terão que instalar esta atualização de segurança, mas somente uma vez.

**Tenho uma versão não vulnerável do software instalada. Por que eu estou recebendo a oferta desta atualização?**  
Alguns dos softwares não afetados, incluindo o OneNote 2003, Pacote de compatibilidade do Microsoft Office para formatos de arquivo do Word, Excel e PowerPoint 2007, Office Word Viewer 2003, Microsoft Office Excel Viewer, Microsoft Office PowerPoint Viewer 2007, Visual Studio 2008, Expression Web e Expression Web 2 contêm os componentes vulneráveis compartilhados do Office, mas como eles não acessam o código vulnerável, eles não são afetados por esta vulnerabilidade. No entanto, uma vez que o código vulnerável está presente, esta atualização será oferecida.

**A oferta para atualizar uma versão não vulnerável do software representa uma falha no mecanismo de atualização da Microsoft?**  
Não, o mecanismo de atualização está funcionando perfeitamente já que detecta uma versão mais antiga dos arquivos no sistema do que a versão do pacote de utilização e, portanto, oferece a atualização.

**Por que esta atualização é classificada como Crítica para o OneNote 2007 e OneNote 2007 Service Pack 1, mas classificada como Importante para outro software afetado?**  
Embora esta atualização de segurança elimine a vulnerabilidade, atualizando os arquivos usados na funcionalidade compartilhada do Office, para a vulnerabilidade ser explorada, o OneNote 2007 deve ser instalado e o usuário ainda terá que clicar em um URL do OneNote especialmente criado.

**Uso o Microsoft Office 2003 Service Pack 2. Há recursos de segurança adicionais incluídos nesta atualização?**  
Sim, como parte do modelo de serviço do Microsoft Office 2003, quando os usuários de Microsoft Office 2003 Service Pack 2 instalam esta atualização, seus sistemas são atualizados para a funcionalidade de segurança que foi inicialmente lançada com o Microsoft Office 2003 Service Pack 3. Todas as atualizações lançadas depois de 1 de janeiro de 2008 do Microsoft Office 2003 Service Pack 2 incluirão estes recursos de segurança, que foram introduzidos no [Microsoft Office 2003 Service Pack 3](http://www.microsoft.com/downloads/details.aspx?familyid=e25b7049-3e13-433b-b9d2-5e3c1132f206&displaylang=en). Nós testamos totalmente essa atualização, mas, como ocorre com todas as atualizações, recomendamos que os usuários façam os testes apropriados em seu ambiente e nas configurações de seus sistemas. Para obter mais informações sobre esse problema, consulte o [Artigo 951646 (em inglês) da Base de Conhecimento Microsoft](http://support.microsoft.com/kb/951646).

Vulnerabilidade de erro na validação do localizador uniforme de recursos - CVE-2008-3007
----------------------------------------------------------------------------------------


Existe uma vulnerabilidade de execução remota de código na maneira como o Microsoft Office manipula URLs especialmente criados usando o manipulador de protocolo do OneNote (onenote://). A vulnerabilidade pode permitir a execução remota de código se um usuário clicar em um URL do OneNote especialmente criado. O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário.

Para exibir essa vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2008-3007](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3007).

#### Fatores atenuantes para a vulnerabilidade de erro na validação do localizador uniforme de recursos - CVE-2008-3007

A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:

-   No cenário de ataque na Web, um site pode conter um link especialmente criado (onenote://) usado para explorar esta vulnerabilidade. Um invasor tem que persuadir os usuários a visitar o site e abrir um URL do OneNote especialmente criado, geralmente fazendo com que eles cliquem em um link em um email ou mensagem do Instant Messenger que leva os usuários ao site do invasor e convencê-los a abrir o link especialmente criado.
-   O invasor que explorar com êxito a vulnerabilidade poderá obter os mesmos direitos que o usuário local. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.
-   A vulnerabilidade não pode ser explorada automaticamente com a visualização de um email. Para que um ataque seja bem-sucedido, um usuário deve clicar em um link especialmente criado enviado em uma mensagem de email.

#### Soluções alternativas para vulnerabilidade de erro na validação do localizador uniforme de recursos - CVE-2008-3007

A solução alternativa refere-se a uma configuração ou alteração de configuração que não corrige a vulnerabilidade subjacente mas que ajudaria a bloquear vetores de ataque conhecidos antes de aplicar a atualização. A Microsoft testou as soluções alternativas e estados a seguir como parte do cenário que implica se uma solução alternativa reduziria funcionalidade:

-   **Desativar o manipulador de protocolo OneNote**

    **Observação** Reparar o Office ou instalar uma atualização de segurança do Office poderá desfazer essa solução alternativa.

    **Aviso** O uso incorreto do Editor do Registro pode causar problemas graves e exigir a reinstalação do sistema operacional. A Microsoft não garante a solução de problemas resultantes do uso incorreto do Editor do Registro. Você é responsável pelo uso do Editor do Registro. Para obter informações sobre como editar o Registro, consulte o tópico da Ajuda "Alterar chaves e valores" do Editor do Registro (Regedit.exe) ou exiba os tópicos da Ajuda "Adicionar e excluir informações no Registro" e "Editar informações do Registro" no Regedt32.exe.

    **Observação** É recomendável fazer backup do Registro antes de editá-lo.

    Para desabilitar o manipulador de protocolo, siga estas etapas:

    **Método Interativo**

    1.  Clique em **Iniciar**, em **Executar**, digite "regedit" (sem as aspas) e clique em **OK**.
    2.  Expanda **HKEY\_CLASSES\_ROOT**, clique em **OneNote** e, em seguida, clique no menu de Registro e selecione **Exportar**.
    3.  Na caixa de diálogo Exportar Arquivo do Registro, digite "OneNote registry backup.reg" e pressione **Salvar**. Isto criará um backup desta chave do Registro na pasta 'Meus Documentos’ por padrão.
    4.  Expanda a chave do **OneNote**, clique no valor do Registro de Protocolo de URL no painel direito e pressione a tecla **Delete**. Quando solicitado a excluir o valor de Registro pela caixa de diálogo Confirmar a Exclusão do Valor, selecione **Sim**.

    **Script para implantação gerenciada**

    1.  Primeiro pode ser feita uma cópia de backup das chaves do Registro de um script de implantação gerenciado com os seguintes comandos:
        Regedit.exe /e OneNote\_registry\_backup.reg HKEY\_CLASSES\_ROOT\\OneNote
    2.  Em seguida, salve o seguinte em um arquivo com a extensão .REG (ou seja, Delete\_OneNote\_URL\_Protocol.reg):
        Windows Registry Editor Versão 5.00
        \[HKEY\_CLASSES\_ROOT\\OneNote\]
        "URL Protocol"=-
    3.  Execute o script de Registro criado na etapa 2 na máquina de destino com o seguinte comando:
        Regedit.exe /s Delete\_OneNote\_URL\_Protocol.reg

    **Impacto da solução alternativa.**

    Esta solução alternativa desabilita o manipulador de protocolo do OneNote.

    **Como desfazer a solução alternativa.**

    Restaure a chave do Registro usando Regedit para restaurar as configurações salvas no arquivo .REG.

-   **Redirecione o manipulador de protocolo do OneNote para o manipulador de protocolo Sobre**

    **Observação** Reparar o Office ou instalar uma atualização de segurança do Office poderá desfazer essa solução alternativa.

    **Aviso** O uso incorreto do Editor do Registro pode causar problemas graves e exigir a reinstalação do sistema operacional. A Microsoft não garante a solução de problemas resultantes do uso incorreto do Editor do Registro. Você é responsável pelo uso do Editor do Registro. Para obter informações sobre como editar o Registro, consulte o tópico da Ajuda "Alterar chaves e valores" do Editor do Registro (Regedit.exe) ou exiba os tópicos da Ajuda "Adicionar e excluir informações no Registro" e "Editar informações do Registro" no Regedt32.exe.

    **Observação** É recomendável fazer backup do Registro antes de editá-lo.

    **Método Interativo**

    1.  Clique em **Iniciar**, em **Executar**, digite "regedit" (sem as aspas) e clique em **OK**.
    2.  Expanda **HKEY\_CLASSES\_ROOT\\PROTOCOLS** e clique no **Manipulador**.
    3.  No menu Editar, clique em **Novo** e clique em **Chave**.
    4.  Na caixa de diálogo de nova chave, digite "onenote" (sem as aspas) e pressione **Enter**.
    5.  Expanda a nova chave do Registro **HKEY\_CLASSES\_ROOT\\PROTOCOLS\\Handler\\onenote** e, no menu Editar, clique em **Novo** e, em seguida, em **StringValue**.
    6.  Na caixa de diálogo do novo valor de seqüência de caracteres, digite "CLSID" (sem as aspas).
    7.  No painel correto do Editor do Registro, clique duas vezes no novo valor do Registro "CLSID" e digite ou cole o seguinte na caixa de diálogo "Dados do Valor": {3050F406-98B5-11CF-BB82-00AA00BDCE0B} e pressione o botão **OK**.

    **Script para implantação gerenciada**

    1.  Salve o seguinte em um arquivo com a extensão .REG (ou seja, OneNote\_Handler.reg):
        Windows Registry Editor Versão 5.00
        \[HKEY\_CLASSES\_ROOT\\PROTOCOLS\\Handler\\onenote\]
        "CLSID"="{3050F406-98B5-11CF-BB82-00AA00BDCE0B}"
    2.  Execute o script de Registro criado na etapa 1 na máquina de destino com o seguinte comando:
        Regedit.exe /s OneNote\_Handler.reg

    **Impacto da solução alternativa.**

    Esta solução alternativa redireciona o manipulador de protocolo do OneNote para o manipulador de protocolo Sobre.

    **Como desfazer a solução alternativa.**

-   Interativo
    1.  Clique em **Iniciar**, em **Executar**, digite "regedit" (sem as aspas) e clique em **OK**.
    2.  Expanda **HKEY\_CLASSES\_ROOT\\PROTOCOLS\\Handler**
    3.  Clique na chave do Registro do onenote e pressione a tecla **Delete**.
    4.  Quando solicitado a excluir o valor de Registro pela caixa de diálogo Confirmar Exclusão da Chave, selecione **Sim**.
-   Script para implantação gerenciada
    1.  Salve o seguinte em um arquivo com a extensão .REG (ou seja, Delete\_OneNote\_Handler.reg):
        Windows Registry Editor Versão 5.00
        \[-HKEY\_CLASSES\_ROOT\\PROTOCOLS\\Handler\\onenote\]
    2.  Execute o script de Registro criado na etapa 2 na máquina de destino com o seguinte comando:
        Regedit.exe /s Delete\_OneNote\_Handler.reg

#### Perguntas freqüentes sobre a vulnerabilidade de erro na validação do localizador uniforme de recursos - CVE-2008-3007

**Qual é o escopo da vulnerabilidade?**  
Essa é uma vulnerabilidade de execução remota de código. O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário.

**O que provoca a vulnerabilidade?**  
A vulnerabilidade é causada por um erro de validação que ocorre quando um URL especialmente criado é transmitido para abrir um arquivo do OneNote especialmente criado.

**Para que um invasor pode usar a vulnerabilidade?**  
O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado. Se um usuário estiver conectado com direitos administrativos, o invasor poderá assumir o controle total do sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.

**De que forma o invasor pode explorar a vulnerabilidade?**  
Em um cenário de ataque baseado na Web, um invasor pode convencer um usuário a clicar em um localizador uniforme de recursos especialmente criado que se refere a um arquivo do OneNote especialmente criado que leva à execução remota de código.

**Qual componente é afetado pela vulnerabilidade?**  
O Office OneNote 2007 é uma agenda digital que fornece às pessoas um lugar para reunir suas anotações e informações, uma poderosa ferramenta de pesquisa para encontrar o que procuram rapidamente e agendas compartilhadas fáceis de usar para que possam gerenciar o carregamento de informações e trabalhar com mais eficiência.

**Quais são os sistemas que mais correm riscos com a vulnerabilidade?**  
Os sistemas nos quais o software afetado é usado, como estações de trabalhos e servidores de terminal, são os que correm mais risco. Os servidores correm mais riscos se os administradores permitirem que os usuários façam logon em servidores e executem programas. Entretanto, as práticas recomendadas não recomendam esse procedimento.

**O que a atualização faz?**  
A atualização elimina a vulnerabilidade, modificando a maneira como o Microsoft Office valida os localizadores uniformes de recursos.

**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**  
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes e não testemunhou nenhum exemplo de publicação de código de verificação de conceito, quando este boletim de segurança foi lançado pela primeira vez.

**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**  
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes e não testemunhou nenhum exemplo de publicação de código de verificação de conceito, quando este boletim de segurança foi lançado pela primeira vez.

-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte o tópico "Microsoft Baseline Security Analyzer" na seção **Orientação e ferramentas de detecção e implantação** anteriormente neste boletim para obter mais informações.
  
-   **Verificação de Versão do Arquivo**
  
    Como existem várias versões e edições do Microsoft Windows, as etapas a seguir podem ser diferentes no seu computador. Se isso acontecer, consulte a documentação do produto para completar estas etapas.
  
    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.  
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.  
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.  
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.  
        **Observação** Dependendo da versão do sistema operacional ou dos programas instalados, alguns dos arquivos listados na tabela de informações de arquivos talvez não estejam instalados.  
    5.  Na guia **Versão**, determine a versão do arquivo instalado no sistema, comparando-o com a versão documentada na tabela de informações de arquivos correta.  
        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.
