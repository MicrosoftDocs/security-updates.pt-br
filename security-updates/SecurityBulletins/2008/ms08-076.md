Security Bulletin

Microsoft Security Bulletin MS08-076 - Importante
=================================================

Vulnerabilidades no Windows Media Components podem permitir a execução remota de código (959807)
------------------------------------------------------------------------------------------------

Publicado: terça-feira, 9 de dezembro de 2008 | Atualizado: terça-feira, 24 de novembro de 2009

**Versão:** 5.0

### Informações Gerais

#### Sinopse

Esta atualização de segurança elimina duas vulnerabilidades relatadas em particular nos seguintes componentes do Windows Media: Windows Media Player, Tempo de Execução do Windows Media Format e Windows Media Services. A vulnerabilidade mais grave pode permitir a execução remota de código. Se um usuário tiver feito logon com direitos administrativos, o invasor que explorar com êxito essa vulnerabilidade poderá ter o controle total do sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.

Esta atualização de segurança é classificada como importante para Windows Media Player 6.4, Tempo de Execução do Windows Media Format 7.1, Tempo de Execução do Windows Media Format 9.0, Tempo de Execução do Windows Media Format 9.5, Tempo de Execução do Windows Media Format 11, Windows Media Services 4.1, Windows Media Services 9 Series e Windows Media Services 2008. Para obter mais informações, consulte a subseção **Software afetado e não afetado** nesta seção.

A atualização de segurança elimina a primeira vulnerabilidade, modificando a maneira que as respostas de autenticação do Windows Media são validadas. A atualização de segurança elimina a segunda vulnerabilidade, garantindo que os clientes do Windows Media tratem os servidores usando endereços ISATAP como externos. Para obter mais informação sobre as vulnerabilidades, consulte a subseção Perguntas frequentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A Microsoft recomenda que os clientes instalem a atualização o mais rápido possível.

**Problemas conhecidos.** Nenhuma

#### Software afetado e não afetado

O software a seguir foi testado para determinar quais versões ou edições foram afetadas. O ciclo de vida do suporte das outras versões ou edições já terminou ou elas não são afetadas. Para determinar o ciclo de vida do suporte para sua versão ou edição de software, visite o site [Ciclo de vida do Suporte Microsoft.](http://go.microsoft.com/fwlink/?linkid=21742)

Perguntas freqüentes relacionadas a esta atualização de segurança
-----------------------------------------------------------------


**Por que este boletim foi relançado em 24 de novembro de 2009?**  
A Microsoft relançou este boletim para oferecer um novo pacote de atualização específico para os sistemas Windows XP Embedded, com o intuito de solucionar problemas de instalação e detecção nos pacotes de atualização do Windows XP originais incluídos neste boletim.

Além disso, a Microsoft revisou os pacotes de atualização existentes para o Windows XP Service Pack 2 (KB952069) e o Windows XP Service Pack 3 (KB952069) para impedir que os pacotes sejam oferecidos aos sistemas Windows XP Embedded. Os clientes que já instalaram esta atualização de segurança estão protegidos e não precisam tomar nenhuma previdência.

Os clientes que usam o sistema operacional Windows XP Embedded devem aplicar a atualização na primeira oportunidade. Os clientes também podem baixar manualmente a atualização para o [Windows XP Embedded Service Pack 2](http://www.microsoft.com/downloads/details.aspx?familyid=c8f59b79-7d09-4976-a678-9aef5975c186) (KB972187) ou para o [Windows XP Embedded Service Pack 3](http://www.microsoft.com/downloads/details.aspx?familyid=502dcf76-aec3-48bf-930d-0c02b1604cd1) (KB972187). Nenhuma outra atualização é afetada por este relançamento.

**O que é o sistema operacional Windows XP Embedded?**  
O Microsoft Windows XP Embedded é uma versão com componentes do sistema operacional Windows XP Professional que traz o rico conjunto de recursos do Windows XP Professional aos dispositivos incorporados. O Windows XP Embedded é otimizado para o varejo, com recursos de rede incorporados e suporte plug-and-play baseado em padrões. O Windows XP Embedded permite que você conecte uma variedade de periféricos de dispositivos de hospedagem ou varejo em um único sistema POS. Para obter mais informações sobre os sistemas operacionais Windows XP Embedded, visite o [Centro do Desenvolvedor do Microsoft Windows Embedded](http://msdn.microsoft.com/en-us/windowsembedded/default.aspx).

**Por que este boletim foi relançado em 13 de janeiro de 2009?**  
A Microsoft relançou este boletim para oferecer novos pacotes de atualização para o Tempo de Execução do Windows Media Format 9.5 no Windows XP Service Pack 2 (KB952069) e no Windows XP Service Pack 3 (KB952069). Os novos pacotes de atualização corrigem um problema de instalação que pode resultar em uma instalação incompleta em sistemas Windows XP que executam o Tempo de Execução do Windows Media Format 9.5 em que o cliente instalou o KB944110, uma atualização separada, antes de instalar o KB952069. Tais clientes ainda estão vulneráveis e precisam baixar e instalar os pacotes de atualização relançados. Nenhum outro pacote de atualização é afetado por este relançamento. Os clientes que estiverem executando todas as outras versões com suporte e afetadas do Windows Media Player, do Tempo de Execução do Windows Media Format e do Windows Media Services que já tiverem aplicado com êxito os pacotes de atualização de segurança originais não precisam tomar nenhuma providência. Os pacotes de atualização de segurança relançados estão disponíveis nos mesmos canais de distribuição que as atualizações de segurança originais, inclusive Atualizações Automáticas, Windows Update e Windows Server Update Services.

A Microsoft também revisou este boletim de forma para listar o Windows Media Player 6.4 e o Windows Media Services 4.1 como afetados em todas as edições do Microsoft Windows 2000 Service Pack 4 (e não apenas no Microsoft Windows 2000 Server Service Pack 4). A mesma atualização se aplica. Não houve novos pacotes de atualização para este sistema operacional. No entanto, os clientes a quem esta atualização – KB954600 para Windows Media Player 6.4 ou KB952068 para Windows Media Services 4.1 – foi oferecida, mas que não a aplicaram, precisam fazê-lo.

**Onde estão os detalhes das informações sobre o arquivo?**  
Informações detalhadas sobre o arquivo podem ser encontradas no [Artigo 959807 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/959807).

**Por que esta atualização elimina várias vulnerabilidades de segurança reportadas?**  
Esta atualização contém suporte para diversas vulnerabilidades porque as modificações necessárias para corrigir esses problemas foram feitas em arquivos relacionados. Em vez de instalar várias atualizações quase iguais, os clientes poderão instalar apenas esta atualização

**Uso uma versão mais antiga do software discutido neste boletim de segurança. O que devo fazer?**  
Os softwares afetados listados neste boletim foram testados para determinar que versões são afetadas. Outras versões passaram seu ciclo de vida de suporte. Para determinar o ciclo de vida do suporte para sua versão de software, visite o site [Ciclo de Vida do Suporte Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

Os clientes que possuem versões anteriores do software devem priorizar a migração para as versões com suporte, a fim de evitar uma possível exposição a vulnerabilidades. Para obter mais informações sobre o ciclo de vida do produto Windows, visite o site [Ciclo de Vida do Suporte Microsoft](http://go.microsoft.com/fwlink/?linkid=21742). Para obter mais informações sobre o período de suporte estendido da atualização de segurança dessas versões ou edições do software, visite o site [Microsoft Product Support Services](http://go.microsoft.com/fwlink/?linkid=33328).

Os clientes que precisarem de suporte adicional para software mais antigo deverão entrar em contato com o representante da equipe de contas da Microsoft, o gerente técnico da conta ou o representante do parceiro Microsoft apropriado para obter opções de suporte personalizadas. Vulnerabilidade de SPN - CVE-2008-3009
--------------------------------------


Existe uma vulnerabilidade de reflexão de credencial nos componentes do Windows Media que pode permitir a um invasor executar código com os mesmos direitos do usuário local ou com as credenciais de distribuição do Windows Media Services. A vulnerabilidade existe devido a pontos fracos nas implementações do nome da entidade de serviço (SPN) dos componentes do Windows Media.

Para exibir essa vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2008-3009](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3009).

#### Fatores atenuantes da vulnerabilidade de SPN - CVE-2008-3009

A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:

-   O Windows Media Services é um componente opcional em todas as plataformas do Windows Server e não é instalado por padrão.
-   Em um cenário de ataque com base na Web, um invasor terá que hospedar um site que contenha uma página da Web usada para tentar explorar essa vulnerabilidade. Não há como um invasor forçar usuários a visitar o site mal-intencionado. Em vez disso, o invasor terá que atraí-los para lá, normalmente fazendo com que cliquem em um link para esse site. Depois de clicar no link, os usuários são instruídos a executarem várias ações. Um ataque pode ocorrer após a execução dessas ações.
-   Para Windows Server 2008 e Windows Server 2008 para sistemas baseados em x64, o Tempo de Execução do Windows Media Format 11 está presente no sistema somente quando a Experiência Desktop está instalada.

#### Soluções alternativas para a vulnerabilidade de SPN - CVE-2008-3009

A Microsoft não identificou nenhuma solução alternativa para esta vulnerabilidade.

#### Perguntas frequentes sobre a vulnerabilidade de SPN - CVE-2008-3009

**Qual é o escopo da vulnerabilidade?**  
Essa é uma vulnerabilidade de execução remota de código. Um invasor que explorar com êxito esta vulnerabilidade pode executar o código com os mesmos privilégios do usuário conectado. Se um usuário tiver feito logon com direitos administrativos, o invasor que explorar com êxito essa vulnerabilidade poderá ter o controle total do sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.  Um invasor que explorar com êxito esta vulnerabilidade também pode executar código com credenciais de distribuição do Windows Media Services. Nessa instância, um invasor pode executar código com os mesmos direitos que a conta de SERVIÇO DE REDE.

**O que provoca a vulnerabilidade?**  
Os componentes do Windows Media não optam corretamente para as proteções de reflexão de credencial NTLM para garantir que as credenciais de um usuário não sejam refletidas de volta e usadas para personificá-lo.

**O que são Nomes de Entidade de Serviço (SPNs)?**  
Um nome de entidade de serviço (SPN) é o nome pelo qual o cliente identifica exclusivamente uma instância de um serviço. O SPN fornece um tipo de identidade de serviço usado para autenticar o serviço ao cliente. Quando um cliente quer se conectar a um serviço, localiza uma instância do serviço, compõe um SPN para aquela instância, se conecta ao serviço e apresenta o SPN para o serviço autenticar. Para obter informações adicionais sobre o SPN, consulte [Identidade e autenticação de serviços](http://msdn.microsoft.com/en-us/library/ms733130.aspx). Para obter informações adicionais sobre como os SPNs devem ser formatados, consulte [Formatos de nome para SPNs exclusivos](http://msdn.microsoft.com/en-us/library/ms677601(vs.85).aspx).

**Para que um invasor pode usar a vulnerabilidade?**  
O invasor que explorar com êxito a vulnerabilidade poderá obter os mesmos direitos que o usuário conectado. Se um usuário tiver feito logon com direitos administrativos, o invasor que explorar com êxito essa vulnerabilidade poderá ter o controle total do sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.

**De que forma o invasor pode explorar a vulnerabilidade?**  
Esta vulnerabilidade requer que um usuário com uma versão afetada dos componentes do Windows Media acesse um servidor mal-intencionado. Um invasor poderia hospedar um site ou um compartilhamento do servidor mal-intencionado que requer autenticação. Quando o cliente se conecta ao servidor mal-intencionado e se autentica, o invasor pode usar as credenciais fornecidas pelo cliente para refletir de volta ao cliente, ou encaminhar as credenciais para outros servidores, imitando o cliente. Não há como um invasor forçar usuários a visitar um site ou compartilhamento de servidor especialmente criado. Em vez disso, o invasor teria que convencê-los a visitar o site ou o compartilhamento de servidor, geralmente fazendo com que eles cliquem em um link em um email ou mensagem do Instant Messenger que leva os usuários ao site do invasor.

**Quais são os sistemas que mais correm riscos com a vulnerabilidade?**  
Qualquer sistema que use as versões afetadas do Windows Media Player, Tempo de Execução do Windows Media Format ou Windows Media Services corre risco com esta vulnerabilidade. Qualquer sistema em que esses componentes sejam usados com frequência, como estações de trabalho ou servidores de terminal, está mais exposto a essa vulnerabilidade.

**O que a atualização faz?**  
A atualização de segurança elimina a vulnerabilidade modificando a maneira como as respostas de autenticação do Windows Media Component são validadas para impedir a repetição de credenciais.

**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**  
Não. A Microsoft recebeu informações sobre essa vulnerabilidade por meio de divulgação responsável. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido divulgada publicamente quando este boletim de segurança foi lançado pela primeira vez.

**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**  
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes e não testemunhou nenhum exemplo de publicação de código de verificação de conceito, quando este boletim de segurança foi lançado pela primeira vez.

Vulnerabilidade de ISATAP - CVE-2008-3010
-----------------------------------------


Existe uma vulnerabilidade de divulgação de informações nas versões com suporte dos componentes do Windows Media que podem resultar na divulgação de credenciais NTLM. Qualquer componente do Windows Media que acesse um URL que usa um endereço ISATAP pode vazar as credenciais NTLM do usuário para o servidor que hospeda o URL. Isso pode permitir a um invasor externo à zona de intranet reunir credenciais NTLM de um ambiente de empresa.

Para exibir essa vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2008-3010](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3010).

#### Fatores atenuantes da vulnerabilidade de ISATAP - CVE-2008-3010

A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:

-   As práticas recomendadas de firewall e as configurações de firewall padrão podem ajudar a proteger recursos de rede contra ataques com origem externa ao perímetro da empresa. Essas práticas também recomendam que os sistemas conectados à Internet tenham um número mínimo de portas e protocolos expostos.
-   Em um cenário de ataque com base na Web, um invasor terá que hospedar um site que contenha uma página da Web usada para tentar explorar essa vulnerabilidade. Não há como um invasor forçar usuários a visitar o site mal-intencionado. Em vez disso, o invasor terá que atraí-los para lá, normalmente fazendo com que cliquem em um link para esse site. Depois de clicar no link, os usuários são instruídos a executarem várias ações. Um ataque pode ocorrer após a execução dessas ações.
-   Para Windows Server 2008 e Windows Server 2008 para sistemas baseados em x64, o Tempo de Execução do Windows Media Format 11 está presente no sistema somente quando a Experiência Desktop está instalada.

#### Soluções alternativas para a vulnerabilidade de ISATAP - CVE-2008-3010

A solução alternativa refere-se a uma configuração ou alteração de configuração que não corrige a vulnerabilidade subjacente mas que ajudaria a bloquear vetores de ataque conhecidos antes de aplicar a atualização. A Microsoft testou as soluções alternativas e estados a seguir como parte do cenário que implica se uma solução alternativa reduziria funcionalidade:

-   **Bloquear o Tipo de Protocolo IP 41 (ISATAP) no firewall**

    O protocolo ISATAP é definido como tipo de protocolo 41 por [RFC4214: ISATAP](http://www.ietf.org/rfc/rfc4214.txt). O bloqueio desse tipo de protocolo no firewall ajudará a proteger os sistemas atrás do firewall contra as tentativas de exploração dessa vulnerabilidade. A Microsoft recomenda que você bloqueie todas as comunicações de entrada não solicitadas da Internet para ajudar a impedir ataques que talvez usem outras portas ou protocolos.

#### Perguntas frequentes sobre a vulnerabilidade de ISATAP - CVE-2008-3010

**Qual é o escopo da vulnerabilidade?**  
Essa é uma vulnerabilidade de divulgação de informações. O invasor que explorar com êxito a vulnerabilidade poderá obter as mesmas credenciais NTLM do usuário conectado. Essas credenciais podem ser subsequentemente refletidas de volta no sistema do usuário para executar código com os mesmos direitos como o usuário de logon.

**O que provoca a vulnerabilidade?**  
Os componentes afetados do Windows Media tratam de maneira incorreta os recursos de Internet como parte de uma zona de Intranet local ao se conectar a um servidor que esteja usando um endereço ISATAP.

**O que é ISATAP?**  
O protocolo ISATAP fornece conectividade IPv6 em uma intranet IPv4. O RFC para ISATAP é hospedado pelo Internet Engineering Task Force (IETF). Para obter informações sobre o ISATAP, consulte [RFC4214: ISATAP](http://www.ietf.org/rfc/rfc4214.txt).

**O que é NTLM?**  
O NTLM é um protocolo de autenticação baseado em um mecanismo de desafio/resposta.

**Para que um invasor pode usar a vulnerabilidade?**  
Um invasor que explorar com êxito esta vulnerabilidade pode obter as credenciais NTLM do usuário.  Essas credenciais podem ser subsequentemente refletidas de volta no sistema do usuário para executar código com os mesmos direitos como o usuário de logon.

**De que forma o invasor pode explorar a vulnerabilidade?**

**Verificando a aplicação da atualização**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção **Orientação e ferramentas de detecção e implantação** neste boletim para obter mais informações.

-   **Verificação de Versão do Arquivo**

    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.

    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.
        **Observação** Dependendo da edição do sistema operacional, ou dos programas que são instalados em seu sistema, alguns arquivos que estão listados na tabela de informações de arquivo podem não estar instalados.

        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.

#### Informações de implantação

**Instalando a atualização**

Quando você instala esta atualização de segurança, o instalador verifica se alguns dos arquivos que estão sendo atualizados no sistema foram anteriormente atualizados por um hotfix da Microsoft.

Se você já tiver instalado um hotfix para atualizar um desses arquivos, o instalador copiará os arquivos SP2QFE ou SP3QFE para o sistema. Caso contrário, o instalador copiará os arquivos SP2GDR ou SP3GDR para o sistema. As atualizações de segurança podem não conter todas as variações desses arquivos. Para obter mais informações sobre esse comportamento, consulte o [Artigo 824994 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/824994).

Para obter mais informações sobre o instalador, visite o [site Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

Para obter mais informações sobre a terminologia que aparece neste boletim, como *hotfix*, consulte o [Artigo 824684 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/824684).

**Verificando a aplicação da atualização**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção **Orientação e ferramentas de detecção e implantação** neste boletim para obter mais informações.

-   **Verificação de Versão do Arquivo**

    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.

    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.
        **Observação** Dependendo da edição do sistema operacional, ou dos programas que são instalados em seu sistema, alguns arquivos que estão listados na tabela de informações de arquivo podem não estar instalados.
    5.  Na guia **Versão**, determine a versão do arquivo instalado no sistema, comparando-o com a versão documentada na tabela de informações de arquivos correta.
        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.

-   **Verificação da Chave do Registro**

    Você também pode ser capaz de verificar os arquivos que esta atualização de segurança instalou examinando as chaves do Registro na **Tabela de referência** nesta seção.

    Essas chaves do Registro podem não conter uma lista completa dos arquivos instalados. Além disso, talvez essas chaves do Registro não sejam criadas corretamente se um administrador ou um OEM integrar ou adicionar a atualização de segurança aos arquivos fonte da instalação do Windows.

#### Informações de implantação

**Instalando a atualização**

Quando você instala esta atualização de segurança, o instalador verifica se alguns dos arquivos que estão sendo atualizados no computador foram anteriormente atualizados por um hotfix da Microsoft.

Se você já tiver instalado um hotfix para atualizar um desses arquivos, o instalador copiará os arquivos SP1QFE ou SP2QFE para o sistema. Caso contrário, o instalador copiará os arquivos SP1GDR ou SP2GDR para o sistema. As atualizações de segurança podem não conter todas as variações desses arquivos. Para obter mais informações sobre esse comportamento, consulte o [Artigo 824994 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/824994).

Para obter mais informações sobre o instalador, visite o [site Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

Para obter mais informações sobre a terminologia que aparece neste boletim, como *hotfix*, consulte o [Artigo 824684 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/824684).
