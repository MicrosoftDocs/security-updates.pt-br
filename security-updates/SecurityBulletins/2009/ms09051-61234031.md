Security Bulletin

Microsoft Security Bulletin MS09-051 - Crítica
==============================================

Vulnerabilidades no Tempo de Execução do Windows Media podem permitir a execução remota de código (975682)
----------------------------------------------------------------------------------------------------------

Publicado: terça-feira, 13 de outubro de 2009 | Atualizado: terça-feira, 10 de novembro de 2009

**Versão:** 2.0

### Informações Gerais

#### Sinopse

Esta atualização de segurança elimina duas vulnerabilidades relatadas em particular no Tempo de execução do Windows Media. As vulnerabilidades podem permitir a execução remota de código se um usuário abrir um arquivo de mídia especialmente criado ou conteúdo de fluxo especialmente criado recebido de um site ou de qualquer aplicativo que entregue conteúdo da Web. O invasor que explorar com êxito as vulnerabilidades poderá obter os mesmos direitos que o usuário local. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.

Esta atualização de segurança é classificada como Crítica para Codec de voz do DirectShow WMA, Decodificador de voz de áudio do Windows Media e Gerenciador de Compactação de Áudio em edições com suporte do Microsoft Windows 2000; Windows XP; Windows Server 2003, com exceção das edições baseadas no Itanium; Windows Vista e Windows Server 2008, com exceção das edições baseadas no Itanium. Para obter mais informação, consulte a subseção **Software afetado e não afetado** nesta seção.

A atualização de segurança elimina as vulnerabilidades mudando a maneira como o Tempo de Execução do Windows Media processa arquivos ASF e inicializa funções em arquivos de áudio compactados. Para obter mais informação sobre as vulnerabilidades, consulte a subseção Perguntas frequentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A maioria dos clientes tem o recurso de atualizações automáticas habilitado e não precisará tomar nenhuma providência porque esta atualização de segurança será baixada e instalada automaticamente. Os clientes que não têm o recurso de atualizações automáticas habilitado precisarão verificar as atualizações e instalar esta atualização manualmente. Para obter informações sobre opções de configuração específicas em atualizações automáticas, consulte o [Artigo 294871 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Para instalações feitas por administradores ou empresas, ou por usuários finais que desejam instalar esta atualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a atualização imediatamente usando software de gerenciamento de atualização ou verificando se há atualizações no serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a seção **Orientação e ferramentas de detecção e implantação** mais adiante neste boletim.

**Problemas conhecidos.** Nenhuma

Para determinar o ciclo de vida do suporte para sua versão ou edição de software, visite o site [Ciclo de vida do Suporte Microsoft.](http://go.microsoft.com/fwlink/?linkid=21742)

Perguntas freqüentes relacionadas a esta atualização de segurança  
-----------------------------------------------------------------
  
<span></span>
**Por que este boletim foi relançado em 10 de novembro de 2009?**    
A Microsoft relançou este boletim para voltar a oferecer a atualização para o Gerenciador de Compactação de Áudio no Microsoft Windows 2000 Service Pack 4, com o objetivo de corrigir um problema de detecção. Esta é apenas uma alteração de detecção; não houve nenhuma alteração aos binários. Os clientes que atualizaram seus sistemas com êxito não necessitam reinstalar esta atualização.
  
**Onde estão os detalhes das informações sobre o arquivo?**    
Consulte as tabelas de referência na seção **Implantação de atualização de segurança** para o local dos detalhes de informações sobre o arquivo.
  
**Qual a relação entre este boletim e o MS09-052, Vulnerabilidade no Windows Media Player pode permitir a execução remota de código?**    
Este boletim não está relacionado ao boletim [MS09-052](http://technet.microsoft.com/security/bulletin/ms09-052), Vulnerabilidade no Windows Media Player pode permitir a execução remota de código. Os usuários devem instalar ambas as atualizações para garantir que eles estejam totalmente protegidos.
  
**Tenho o Windows Media Player instalado em meu computador. Por que não estão me oferecendo algumas das atualizações? **  
Estas atualizações serão oferecidas automaticamente aos sistemas somente se estes tiverem instalado o Windows Media Player, assim como os codecs de áudio opcionais afetados por estas vulnerabilidades. Os codecs associados a KB969878 aplicam-se somente a sistemas de 32 bits. Os codecs associados a KB954155 não são instalados no Windows Media Format SDK baseado em x64, por padrão, mas podem ser baixados automaticamente pelo Windows Media Player ou manualmente do [Centro de Download da Microsoft](http://www.microsoft.com/downloads/en/default.aspx). Se você não tiver os codecs instalados e tentar instalar a atualização manualmente, poderá receber uma das seguintes mensagens de erro:
  
-   O Windows Media Player 9 é necessário para instalar esta atualização.  
-   O Windows Media Format 9.5 SDK precisa estar na sua máquina para que esta atualização possa ser instalada.
  
Se você receber esta mensagem, esta atualização não se aplica ao seu sistema.  Estas mensagens somente ocorrerão se você tentar instalar a atualização manualmente.
  
**Por que esta atualização elimina várias vulnerabilidades de segurança reportadas?**    
Esta atualização contém suporte para diversas vulnerabilidades porque as modificações necessárias para corrigir esses problemas foram feitas em arquivos relacionados. Em vez de instalar várias atualizações quase iguais, os clientes poderão instalar apenas esta atualização
  
**O que é o Kit de Desenvolvedores de Software (SDK) do Windows Media Format?**    
O Windows Media Format SDK fornece aos desenvolvedores de aplicativos acesso aos componentes do Windows Media Format. O propósito principal do Windows Media Format SDK é capacitar os desenvolvedores a criar aplicativos que reproduzem, gravam, editam, criptografam e entregam arquivos ASF (Advanced Systems Format) e fluxos de rede. Esses arquivos e fluxos comumente contêm conteúdo de áudio e vídeo codificado com os codecs de áudio e vídeo do Windows Media. No entanto, ASF pode conter qualquer tipo de dados. Para obter mais informações, consulte a [página tecnológica do Windows Media Format SDK](http://msdn.microsoft.com/en-us/windows/bb190326.aspx).
  
**Uso uma versão mais antiga do software discutido neste boletim de segurança. O que devo fazer?**    
Os softwares afetados listados neste boletim foram testados para determinar que versões são afetadas. Outras versões passaram seu ciclo de vida de suporte. Para determinar o ciclo de vida do suporte para sua versão de software, visite o site [Ciclo de Vida do Suporte Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).
  
Os clientes que possuem versões anteriores do software devem priorizar a migração para as versões com suporte, a fim de evitar uma possível exposição a vulnerabilidades. Para obter mais informações sobre o ciclo de vida do produto Windows, visite o site [Ciclo de Vida do Suporte Microsoft](http://go.microsoft.com/fwlink/?linkid=21742). Para obter mais informações sobre o período de suporte estendido da atualização de segurança dessas versões ou edições do software, visite o site [Microsoft Product Support Services](http://go.microsoft.com/fwlink/?linkid=33328).
  
Os clientes que precisarem de suporte adicional para software mais antigo deverão entrar em contato com o representante da equipe de contas da Microsoft, o gerente técnico da conta ou o representante do parceiro Microsoft apropriado para obter opções de suporte personalizadas. Os clientes sem um contrato Alliance, Premier ou Authorized podem entrar em contato com o escritório de vendas local da Microsoft. Para obter informações de contato, visite o site de [Informações internacionais da Microsoft](http://go.microsoft.com/fwlink/?linkid=33329), selecione o país e clique em **Go** para ver uma lista de telefones. Ao ligar, peça para falar com o gerente de vendas local de Suporte Premier. Para obter mais informações, consulte a seção [Perguntas frequentes sobre o ciclo de vida do suporte ao produto sistema operacional Windows](http://go.microsoft.com/fwlink/?linkid=33330).

Vulnerabilidade na taxa de amostragem de voz do Tempo de Execução do Windows Media - CVE-2009-0555  
--------------------------------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código no Windows Media Player devido ao processamento indevido de arquivos ASF (Advanced Systems Format) especialmente criados. Um invasor pode explorar a vulnerabilidade criando um arquivo de áudio especial, que pode permitir a execução remota de código quando reproduzido por uma versão afetada do Windows Media Player. O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado.
  
Para exibir essa vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2009-0555](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0555).
  
#### Fatores atenuantes da Vulnerabilidade na taxa de amostragem de voz do Tempo de Execução do Windows Media - CVE-2009-0555
  
A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:
  
-   No cenário de ataque na Web, o invasor terá que hospedar um site contendo uma página da Web usada para explorar essa vulnerabilidade. Além disso, sites comprometidos e sites que aceitem ou hospedem conteúdo fornecido pelo usuário ou anúncios pode conter conteúdo especialmente desenvolvido que pode explorar esta vulnerabilidade. Em todos os casos, entretanto, um invasor não teria como forçar os usuários a visitarem os sites. Em vez disso, o invasor teria que persuadir os usuários a visitar o site, geralmente fazendo com que eles cliquem em um link em um email ou mensagem do Instant Messenger que leva o usuário ao site do invasor.  
-   O invasor que explorar com êxito a vulnerabilidade poderá obter os mesmos direitos que o usuário local. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.  
-   O Windows Server 2008 não é afetado, a menos que o componente opcional de Experiência Desktop também esteja instalado.
  
#### Soluções alternativas para a Vulnerabilidade na taxa de amostragem de voz do Tempo de Execução do Windows Media - CVE-2009-0555
  
A solução alternativa refere-se a uma configuração ou alteração de configuração que não corrige a vulnerabilidade subjacente mas que ajudaria a bloquear vetores de ataque conhecidos antes de aplicar a atualização. A Microsoft testou as soluções alternativas e estados a seguir como parte do cenário que implica se uma solução alternativa reduziria funcionalidade:
  
-   **Cancelar o registro de wmspdmod.dll**
  
    Para cancelar o registro desta DLL, execute o comando a seguir em um prompt de comando elevado.
  
    Para sistemas baseados em Windows de 32 bits:
  
    `Regsvr32.exe –u %WINDIR%\system32\wmspdmod.dll`
  
    Para sistemas Windows baseados em x64:
  
    `Regsvr32.exe –u %WINDIR%\syswow64\wmspdmod.dll`
  
    **Impacto da solução alternativa.** Os fluxos de áudio em arquivos ASF que usam o Código de Fala do Windows Media podem não ser reproduzidos.
  
    **Como desfazer a solução alternativa.**
  
    Execute o comando a seguir em um prompt de comando elevado.
  
    Para sistemas baseados em Windows de 32 bits:
  
    `Regsvr32.exe %WINDIR%\system32\wmspdmod.dll`
  
    Para sistemas Windows baseados em x64:
  
    `Regsvr32.exe %WINDIR%\syswow64\wmspdmod.dll`
  
#### Perguntas frequentes sobre a Vulnerabilidade na taxa de amostragem de voz do Tempo de Execução do Windows Media - CVE-2009-0555
  
**Qual é o escopo da vulnerabilidade?**    
Essa é uma vulnerabilidade de execução remota de código. Se um usuário tiver feito logon com direitos administrativos, o invasor que explorar com êxito essa vulnerabilidade poderá ter o controle total do sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.
  
**O que provoca a vulnerabilidade?**    
O Tempo de Execução do Windows Media processa incorretamente arquivos ASF especialmente criados que utilizam o codec Windows Media Speech.
  
**O que é Tempo de Execução do Windows Media Format?**    
O Tempo de Execução do Microsoft Windows Media Format fornece informações e ferramentas para aplicativos que usam o conteúdo do Windows Media.
  
**O que é ASF (Advanced Systems Format)?**    
ASF (Advanced Systems Format) é um formato de arquivo que armazena informações de áudio e vídeo e é especialmente projetado para execução em redes como a Internet. É um formato compactado que pode conter fluxo contínuo de áudio, vídeo, apresentações de slides e eventos sincronizados. O ASF permite que conteúdo seja entregue como um fluxo contínuo de dados. Os arquivos ASF podem ter a extensão de arquivo ASF, WMV ou WMA.
  
ASF se refere ao formato subjacente do arquivo de áudio. Os arquivos ASF podem ter muitas extensões diferentes de arquivo, sendo que os mais comuns são WMA e WMV. Os arquivos ASF também podem ser fornecidos com outras extensões de arquivo, como MP3, que não refletem o formato verdadeiro do arquivo. O Windows Media Player analisará o formato real do arquivo e ainda tentará reproduzir o arquivo.
  
**O que é o Decodificador de voz de áudio do Windows Media?**    
O Decodificador de voz de áudio do Windows Media decodifica fluxos que foram codificados pelo [Codificador de Áudio do Windows Media](http://msdn.microsoft.com/en-us/library/dd756600(vs.85).aspx). O codec de voz de áudio do Windows Media fornece compactação com baixa taxa de bits, otimizada para áudio que contém fala. A capacidade do codec de produzir amostras tão pequenas se deve ao intervalo limitado de frequência dos sons da voz humana. Essa otimização significa que um codificador dedicado de voz cria produção de baixa qualidade para conteúdo que contém sons mais complicados, como música. No entanto, o codec de voz de áudio do Windows Media compensa esse problema potencial de qualidade fornecendo modos separados para voz, música e conteúdo misto. O codec analisa conteúdo misto para determinar que modo usar em cada parte do arquivo. Para obter mais informações sobre o decodificador de áudio do Windows Media, consulte o artigo do MSDN, [Using the Windows Media Audio Voice Codec](http://msdn.microsoft.com/en-us/library/dd443456(vs.85).aspx).
  
**Para que um invasor pode usar a vulnerabilidade?**    
Um invasor que explorar com êxito essa vulnerabilidade poderá executar código arbitrário. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário.
  
**De que forma o invasor pode explorar a vulnerabilidade?**    
Esta vulnerabilidade requer que um usuário abra um arquivo ASF especialmente criado com uma versão afetada do Windows Media Player.
  
Em um cenário de ataque com base na Web, o invasor terá que hospedar um arquivo ASF especialmente criado usado para tentar explorar esta vulnerabilidade. Não há como um invasor forçar usuários a visitar o site especialmente criado. Em vez disso, o invasor teria que convencê-los a visitar o site, geralmente fazendo com que eles cliquem em um link em um email ou mensagem do Instant Messenger que leva os usuários ao site do invasor.
  
Em um cenário de ataque por email, o invasor pode explorar a vulnerabilidade enviando para o usuário um arquivo de mídia com um arquivo ASF especialmente criado incorporado e convencendo o usuário a abrir o arquivo.
  
**Quais são os sistemas que mais correm riscos com a vulnerabilidade?**    
Qualquer sistema que use o Tempo de Execução do Windows Media, como estações de trabalho ou servidores de terminal, corre risco com esta vulnerabilidade.
  
**O que a atualização faz?**    
A atualização elimina esta vulnerabilidade mudando a maneira como o Tempo de Execução do Windows Media processa arquivos ASF.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Sim. Essa vulnerabilidade foi divulgada publicamente. Ela recebeu o número [CVE-2009-0555](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-0555) da lista Common Vulnerability and Exposure.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**    
Sim. A Microsoft está ciente dos ataques limitados que tentam explorar a vulnerabilidade.
  
Vulnerabilidade de corrupção de heap do Tempo de Execução do Windows Media - CVE-2009-2525  
------------------------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código na maneira como o Tempo de Execução do Windows Media lida com determinadas funções em arquivos de áudio compactados. Essa vulnerabilidade pode permitir a execução remota de código se o usuário abrir um arquivo especialmente criado. Se um usuário tiver feito logon com direitos administrativos, o invasor que explorar com êxito essa vulnerabilidade poderá ter o controle total do sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Para exibir essa vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2009-2525](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2525).

#### Soluções alternativas para a Vulnerabilidade de corrupção de heap do Tempo de Execução do Windows Media - CVE-2009-2525
  
-   **Negar acesso a msaud32.acm**
  
    Nos sistemas Windows 2000, Windows XP e Windows Server 2003:
  
    Para sistemas baseados em Windows de 32 bits:
  
    `Echo y| cacls %WINDIR%\system32\msaud32.acm /E /P everyone:N`
  
    Para sistemas baseados em Windows de 64 bits:
  
    `Echo y| cacls %WINDIR%\syswow64\msaud32.acm /E /P everyone:N`
  
    No Windows Vista e no Windows Server 2008:
  
    Como administrador, execute os seguintes comandos a partir de um prompt de comando:
  
    Para Windows de 32 bits:
  
    `Takeown.exe /f "%windir%\system32\msaud32.acm"Icacls.exe "%windir%\system32\msaud32.acm " /save %TEMP%\msaud32_acm.TXTIcacls.exe "%windir%\system32\msaud32.acm " /deny everyone:(F)`
  
    Para sistemas baseados em Windows de 64 bits:
  
    `Takeown.exe /f "%windir%\syswow64\msaud32.acm"Icacls.exe "%windir%\syswow64\msaud32.acm " /save %TEMP%\ msaud32_acm.TXTIcacls.exe "%windir%\syswow64\msaud32.acm " /deny everyone:(F)`
  
    **Impacto da solução alternativa:** Os aplicativos que dependem do Gerenciador de Compactação de Áudio (ACM) não conseguirão renderizar conteúdos corretamente
  
    **Como desfazer a solução alternativa:**
  
    Nos sistemas Windows 2000, Windows XP e Windows Server 2003:
  
    Para sistemas baseados em Windows de 32 bits:
  
    `cacls “%WINDIR%\system32\msaud32.acm” /E /R everyone`
  
    Para sistemas baseados em Windows de 64 bits:
  
    `cacls “%WINDIR%\syswow64\msaud32.acm” /E /R everyone`
  
    No Windows Vista e no Windows Server 2008:
  
    Como administrador, execute os seguintes comandos a partir de um prompt de comando:
  
    Para Windows de 32 bits:
  
    `icacls %windir%\syswow64 /restore %TEMP%\msaud32_acm.TXT`
  
    Para sistemas baseados em Windows de 64 bits:
  
    `icacls %windir%\syswow64 /restore %TEMP%\msaud32_acm.TXT`
  
#### Perguntas frequentes sobre a Vulnerabilidade de corrupção de heap do Tempo de Execução do Windows Media - CVE-2009-2525
  
**Qual é o escopo da vulnerabilidade?**    
Essa é uma vulnerabilidade de execução remota de código. O invasor que explorar com êxito essa vulnerabilidade poderá assumir remotamente o controle total de um sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário.
  
**O que provoca a vulnerabilidade?**    
O Tempo de Execução do Microsoft Windows Media não inicializa corretamente determinadas funções em arquivos de áudio compactados.
  
**O que é Tempo de Execução do Windows Media Format?**    
O Tempo de Execução do Microsoft Windows Media Format fornece informações e ferramentas para aplicativos que usam o conteúdo do Windows Media.
  
**O que é o Gerenciador de Compactação de Áudio?**    
O Gerenciador de Compactação de Áudio (ACM) adiciona suporte no nível do sistema para compactação e descompactação transparente de áudio em tempo de execução, bem como processamento de dados de áudio do Waveform. O ACM usa ganchos de interface de driver existentes para substituir o algoritmo de mapeamento padrão para dispositivos de áudio de forma de onda. Isto permite ao ACM interceptar chamadas de dispositivos abertos. Depois que uma chamada tiver sido interceptada, o ACM pode executar uma variedade de tarefas para processar os dados de áudio, como inserir um compactador ou descompactador externo na sequência. Para obter mais informações sobre o ACM, consulte o artigo do MSDN [About the Audio Compression Manager](http://msdn.microsoft.com/en-us/library/dd742878(vs.85).aspx).
  
**Para que um invasor pode usar a vulnerabilidade?**    
Se um usuário tiver feito logon com direitos administrativos, o invasor que explorar com êxito essa vulnerabilidade poderá ter o controle total do sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.
  
**De que forma o invasor pode explorar a vulnerabilidade?**    
A exploração desta vulnerabilidade exige que um usuário abra um arquivo de mídia especialmente criado ou receba conteúdo de fluxo especialmente criado de um site ou de qualquer aplicativo que forneça conteúdo da Web.
  
**Quais são os sistemas que mais correm riscos com a vulnerabilidade?**    
Qualquer sistema que use o Windows Media Player, como estações de trabalho ou servidores de terminal, corre risco com esta vulnerabilidade.
  
**O que a atualização faz?**    
A atualização elimina a vulnerabilidade, corrigindo a maneira como o Tempo de Execução do Microsoft Windows Media inicializa funções em arquivos de áudio compactados.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre essa vulnerabilidade por meio de divulgação responsável.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**    
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes e não testemunhou nenhum exemplo de publicação de código de verificação de conceito, quando este boletim de segurança foi lançado pela primeira vez.

**Verificando a aplicação da atualização**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção **Orientação e ferramentas de detecção e implantação** neste boletim para obter mais informações.

-   **Verificação de Versão do Arquivo**

    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.

    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.
        **Observação** Dependendo da edição do sistema operacional, ou dos programas que são instalados em seu sistema, alguns arquivos que estão listados na tabela de informações de arquivo podem não estar instalados.
    5.  Na guia **Versão**, determine a versão do arquivo instalado no sistema, comparando-o com a versão documentada na tabela de informações de arquivos correta.
        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.

-   **Verificação da Chave do Registro**

    Você também pode ser capaz de verificar os arquivos que esta atualização de segurança instalou examinando as chaves do Registro na **Tabela de referência** nesta seção.

    Essas chaves do Registro podem não conter uma lista completa dos arquivos instalados. Além disso, talvez essas chaves do Registro não sejam criadas corretamente se um administrador ou um OEM integrar ou adicionar a atualização de segurança aos arquivos fonte da instalação do Windows.

#### Informações de implantação

**Instalando a atualização**

Quando você instala esta atualização de segurança, o instalador verifica se alguns dos arquivos que estão sendo atualizados no sistema foram anteriormente atualizados por um hotfix da Microsoft.

Se você já tiver instalado um hotfix para atualizar um desses arquivos, o instalador copiará os arquivos RTMQFE, SP1QFE ou SP2QFE para o sistema. Do contrário, o instalador copiará os arquivos RTMGDR, SP1GDR ou SP2GDR para o sistema. As atualizações de segurança podem não conter todas as variações desses arquivos. Para obter mais informações sobre esse comportamento, consulte o [Artigo 824994 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/824994).

Para obter mais informações sobre o instalador, visite o [site Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

Para obter mais informações sobre a terminologia que aparece neste boletim, como *hotfix*, consulte o [Artigo 824684 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/824684).

**Verificando a aplicação da atualização**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção **Orientação e ferramentas de detecção e implantação** neste boletim para obter mais informações.

-   **Verificação de Versão do Arquivo**

    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.

    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.
        **Observação** Dependendo da edição do sistema operacional, ou dos programas que são instalados em seu sistema, alguns arquivos que estão listados na tabela de informações de arquivo podem não estar instalados.
    5.  Na guia **Versão**, determine a versão do arquivo instalado no sistema, comparando-o com a versão documentada na tabela de informações de arquivos correta.
        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.

-   **Verificação da Chave do Registro**

    Você também pode ser capaz de verificar os arquivos que esta atualização de segurança instalou examinando as chaves do Registro na **Tabela de referência** nesta seção.

    Essas chaves do Registro podem não conter uma lista completa dos arquivos instalados. Além disso, talvez essas chaves do Registro não sejam criadas corretamente se um administrador ou um OEM integrar ou adicionar a atualização de segurança aos arquivos fonte da instalação do Windows.
