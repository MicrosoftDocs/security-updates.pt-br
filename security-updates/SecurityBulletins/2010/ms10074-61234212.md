Microsoft Security Bulletin MS10-074 - Moderada
===============================================

Vulnerabilidade no Microsoft Foundation Classes pode permitir execução remota de código (2387149)
-------------------------------------------------------------------------------------------------

Publicado: terça-feira, 12 de outubro de 2010

**Versão:** 1.0

### Informações Gerais

#### Sinopse

Esta atualização de segurança elimina uma vulnerabilidade divulgada publicamente na Biblioteca MFC. A vulnerabilidade pode permitir a execução remota de código se um usuário tiver efetuado logon com direitos de usuário administrativo e abrir um aplicativo criado com a Biblioteca MFC. O invasor que explorar com êxito esta vulnerabilidade poderá obter as mesmas permissões que o usuário conectado. Se um usuário estiver conectado com direitos administrativos, o invasor poderá assumir o controle total do sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.

Esta atualização de segurança foi classificada como Moderada para todas as versões com suporte do Microsoft Windows. Para obter mais informações, consulte a subseção **Softwares afetados e não afetados**, nesta seção.

A atualização de segurança elimina a vulnerabilidade, adicionando verificações adicionais a solicitações para alterações de título do Windows. Para obter mais informações sobre a vulnerabilidade, consulte a subseção Perguntas frequentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A maioria dos clientes tem o recurso de atualizações automáticas habilitado e não precisará tomar nenhuma providência porque esta atualização de segurança será baixada e instalada automaticamente. Os clientes que não têm o recurso de atualizações automáticas habilitado precisarão verificar as atualizações e instalar esta atualização manualmente. Para obter informações sobre opções de configuração específicas em atualizações automáticas, consulte o [Artigo 294871 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Para instalações feitas por administradores ou empresas ou por usuários finais que desejam instalar esta atualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a atualização de segurança usando software de gerenciamento de atualização ou verificando se há atualizações no serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a seção **Orientação e ferramentas de detecção e implantação** mais adiante neste boletim.

**Problemas conhecidos.** Nenhuma

Vulnerabilidade de estouro de buffer na atualização de títulos de documentos MFC do Windows - CVE-2010-3227  
-----------------------------------------------------------------------------------------------------------
  
Existe uma vulnerabilidade de execução remota de código na maneira como os títulos do Windows são gerenciados em aplicativos gravados usando a Biblioteca MFC. Enquanto a vulnerabilidade está localizada no MFC e está presente nos sistemas operacionais afetados, ela só pode ser explorada se um invasor remoto puder influenciar o título da janela de qualquer janela em um aplicativo MFC. O invasor que explorar esta vulnerabilidade com êxito poderá executar código arbitrário no contexto de segurança do usuário atual.
  
Para exibir esta vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2010-3227](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3227).
  
#### Fatores atenuantes para a vulnerabilidade de estouro de buffer na atualização de títulos de documentos MFC do Windows - CVE-2010-3227
  
A Microsoft não identificou nenhum fator atenuante para essa vulnerabilidade.
  
#### Soluções alternativas para a vulnerabilidade de estouro de buffer na atualização de títulos de documentos MFC do Windows - CVE-2010-3227
  
A Microsoft não identificou nenhuma solução alternativa para esta vulnerabilidade.
  
#### Perguntas frequentes sobre a vulnerabilidade de estouro de buffer na atualização de títulos de documentos MFC do Windows - CVE-2010-3227
  
**Qual é o escopo da vulnerabilidade?**    
Esta vulnerabilidade é uma vulnerabilidade de execução remota de código no contexto de segurança do usuário atualmente conectado.
  
**O que provoca a vulnerabilidade?**    
A vulnerabilidade é causada pela maneira que o MFC manipula as solicitações de aplicativos para renomear os títulos da janela.
  
**O que é a Biblioteca MFC (Microsoft Foundation Class)?**    
A Biblioteca MFC (Microsoft Foundation Class) é uma estrutura de trabalho de aplicativos para programação no Microsoft Windows. Escrito em C++, o MFC fornece a maioria do código necessário para executar as operações padrão como gerenciar janelas, menus e caixas de diálogo, executar entrada/saída básica ou armazenar coleções de objetos de dados.
  
**Esta atualização será oferecida mesmo que eu não tenha vetores de ataque adequados em meu sistema?**    
Sim, esta atualização será oferecida a todos os sistemas que executam software, conforme indicado na seção Softwares afetados. Se nenhum dos aplicativos MFC estiver instalado no momento em um sistema, a Microsoft recomendará que os usuários instalem esta atualização como uma medida de Defesa Profunda, no caso de um vetor de ataque ser introduzido ou se tornar conhecido posteriormente.
  
**Há aplicativos da Microsoft que podem ser usados como um vetor de ataque?**    
A Microsoft pesquisou cuidadosamente todos os aplicativos, mas não achou nenhum aplicativo da Microsoft que possa ser usado com êxito como um vetor de ataque.
  
**A Microsoft está ciente dos aplicativos MFC de terceiros que podem ser usados como um vetor de ataque?**    
Sim. A Microsoft está ciente de um aplicativo de terceiros que pode ser usado como um vetor de ataque. A exploração bem-sucedida desta vulnerabilidade usando este vetor de ataque de terceiros requer que um usuário abra um arquivo especialmente criado com o aplicativo MFC e, em seguida, clique em uma pasta dentro do aplicativo MFC.
  
**Escrevi um aplicativo usando o Microsoft Foundation Classes. Meu aplicativo pode ser usado como um vetor de ataque?**    
Seu aplicativo pode ser um vetor de ataque se todas as condições a seguir forem verdadeiras:
  
-   Seu aplicativo usa a Biblioteca MFC  
-   Seu aplicativo configura títulos de janela com base em informações que podem ser controladas por um usuário remoto
  
**Observação** A Microsoft recomenda que os aplicativos MFC sejam vinculados dinamicamente a versões do MFC enviadas com o Windows. Se seu aplicativo enviar uma versão vulnerável do MFC e ele se vincular a ele de forma estática, será altamente recomendável que você atualize seu aplicativo para que ele se vincule dinamicamente a uma versão do MFC enviada com o Windows.
  
**Para que um invasor pode usar a vulnerabilidade?**    
O invasor que explorar com êxito esta vulnerabilidade poderá obter as mesmas permissões que o usuário conectado. Se um usuário estiver conectado com direitos administrativos, o invasor poderá assumir o controle total do sistema afetado. O invasor poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Os usuários cujas contas são configuradas com poucos direitos de usuário no sistema correm menos riscos do que aqueles que possuem direitos administrativos.
  
**De que forma o invasor pode explorar a vulnerabilidade?**    
Para explorar esta vulnerabilidade com êxito, o sistema de destino precisa ter um aplicativo MFC instalado que possa ser usado como um vetor de ataque. Os aplicativos MFC que podem ser usados como um vetor de ataque devem fornecer uma maneira para que um usuário remoto influencie os títulos das janelas no aplicativo MFC. Os exemplos podem ser uma alteração de título de janela baseada no conteúdo de um arquivo aberto com o aplicativo MFC ou talvez uma alteração de título de janela baseada na entrada de rede como em um aplicativo de bate-papo ou um navegador da Web. Dependendo do vetor de ataque específico, um invasor talvez precise convencer um usuário a abrir um arquivo especialmente criado com um aplicativo MFC específico instalado no sistema do usuário e executar ações específicas. Em um vetor de ataque diferente, como um aplicativo de bate-papo, talvez seja suficiente para o invasor mudar o nome de uma sala de bate-papo ou usar uma identificador especialmente criado ao contatar o usuário.
  
**Quais são os principais sistemas que correm riscos com a vulnerabilidade?**    
As estações de trabalhos e os servidores de terminal são os que mais correm risco se os aplicativos MFC forem instalados, permitindo que um usuário remoto influencie os títulos de janela no aplicativo MFC. Os servidores correm mais riscos se os administradores permitirem que os usuários façam logon em servidores e executem programas MFC. Entretanto, as práticas recomendadas não recomendam esse procedimento.
  
**O que a atualização faz?**    
A atualização elimina a vulnerabilidade, adicionando verificações adicionais a solicitações para alterações de título do Windows.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Sim. Essa vulnerabilidade foi divulgada publicamente. Ela recebeu o número [CVE-2010-3227](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3227) da lista Common Vulnerability and Exposure.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**    
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes quando este boletim de segurança foi lançado pela primeira vez.

**Verificando a aplicação da atualização**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção **Orientação e ferramentas de detecção e implantação** neste boletim para obter mais informações.

-   **Verificação de versão de arquivo**

    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.

    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.
        **Observação** Dependendo da edição do sistema operacional, ou dos programas que são instalados em seu sistema, alguns arquivos que estão listados na tabela de informações de arquivo podem não estar instalados.
    5.  Na guia **Versão**, determine a versão do arquivo instalado no sistema, comparando-o com a versão documentada na tabela de informações de arquivos correta.
        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.
