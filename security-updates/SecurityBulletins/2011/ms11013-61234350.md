Microsoft Security Bulletin MS11-013 - Importante
=================================================

Vulnerabilidades no Kerberos podem permitir elevação de privilégio (2496930)
----------------------------------------------------------------------------

Publicado: terça-feira, 8 de fevereiro de 2011 | Atualizado: quarta-feira, 16 de março de 2011

**Versão:** 2.0

### Informações Gerais

#### Sinopse

Esta atualização de segurança elimina uma vulnerabilidade relatada em particular e uma vulnerabilidade divulgada publicamente no Microsoft Windows. A mais grave dessas vulnerabilidades pode permitir elevação de privilégio se um invasor local autenticado instalar um serviço mal-intencionado em um computador de domínios.

Esta atualização de segurança é classificada como Importante para todas as edições com suporte do Windows XP, Windows Server 2003, Windows 7 e Windows Server 2008 R2. Para obter mais informações, consulte a subseção **Softwares afetados e não afetados**, nesta seção.

Esta atualização elimina as vulnerabilidades, impedindo o uso de algoritmos fracos de hash tanto no Windows Kerberos quando no Windows KDC e impedindo que o cliente faça downgrade do padrão de criptografia para o DES para comunicação Kerberos entre cliente e servidor. Para obter mais informação sobre as vulnerabilidades, consulte a subseção Perguntas frequentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A maioria dos clientes tem o recurso de atualizações automáticas habilitado e não precisará tomar nenhuma providência porque esta atualização de segurança será baixada e instalada automaticamente. Os clientes que não têm o recurso de atualizações automáticas habilitado precisarão verificar as atualizações e instalar esta atualização manualmente. Para obter informações sobre opções de configuração específicas em atualizações automáticas, consulte o [Artigo 294871 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Para instalações feitas por administradores ou empresas, ou por usuários finais que desejam instalar esta atualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a atualização assim que possível usando software de gerenciamento de atualização ou verificando se há atualizações no serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a seção **Orientação e ferramentas de detecção e implantação** mais adiante neste boletim.

**Problemas conhecidos.** Nenhuma

Vulnerabilidade de soma de verificação sem chave do Kerberos - CVE-2011-0043  
----------------------------------------------------------------------------
  
Existe uma elevação de vulnerabilidade de privilégio nas implementações de Kerberos. Existe a vulnerabilidade uma vez que a implementação do Windows Kerberos suporta um mecanismo fraco de hash, que pode permitir que determinados aspectos de um ticket de serviço Kerberos sejam forjados. Um usuário mal-intencionado ou invasor que explorar com êxito esta vulnerabilidade poderá obter um símbolo com privilégios elevados no sistema afetado.
  
Para exibir esta vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2011-0043](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-0043).
  
#### Fatores atenuantes da vulnerabilidade de soma de verificação sem chave do Kerberos - CVE-2011-0043
  
A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:
  
-   Esta vulnerabilidade não é explorável em domínios onde os controladores de domínio executam Windows Server 2008 ou Windows Server 2008 R2.  
-   O invasor precisa ter credenciais de logon válidas e poder fazer logon localmente para explorar essa vulnerabilidade. Essa vulnerabilidade não pode ser explorada remotamente por usuários anônimos.
  
#### Soluções alternativas para a vulnerabilidade de soma de verificação sem chave do Kerberos - CVE-2011-0043
  
A Microsoft não identificou nenhuma solução alternativa para esta vulnerabilidade.
  
#### Perguntas frequentes sobre a vulnerabilidade de soma de verificação sem chave do Kerberos - CVE-2011-0043
  
**Qual é o escopo da vulnerabilidade?**    
Esta é uma vulnerabilidade de elevação de privilégio. Um invasor local autenticado pode executar um serviço mal-intencionado em um sistema que pode enviar um ticket de serviço especialmente criado para obter um token que concederá ao invasor privilégios mais altos nesse sistema.
  
**O que provoca a vulnerabilidade?**    
A vulnerabilidade é causada quando a implementação do Microsoft Kerberos no Microsoft Windows suportar mecanismos fracos de hash como CRC32.
  
**O que é o Kerberos?**    
Kerberos é um protocolo usado mutuamente para autenticar usuários e serviços em uma rede aberta e sem proteção. Permite que os serviços identifiquem corretamente o usuário de uma permissão Kerberos, sem ter que autenticar o usuário no serviço. Isso é feito usando as chaves secretas compartilhadas.
  
O protocolo Kerberos usa chaves secretas compartilhadas para criptografar e assinar credenciais dos usuários. Um cliente é autenticado por um Centro de Distribuição de Chaves Kerberos (KDC). Depois dessa autenticação, o usuário pode solicitar uma permissão de serviço para acessar um serviço específico na rede. Essa permissão inclui a identidade autenticada criptografada do usuário.
  
**Para que um invasor pode usar a vulnerabilidade?**    
O invasor que explorar esta vulnerabilidade com êxito poderá executar código arbitrário no contexto de segurança do sistema local. O invasor poderá então instalar programas; exibir, alterar ou excluir dados ou criar novas contas com esses direitos elevados.
  
**De que forma o invasor pode explorar a vulnerabilidade?**    
Um invasor local autenticado com privilégios para instalar um aplicativo mal-intencionado do Kerberos com privilégios limitados pode explorar esta vulnerabilidade instalando tal serviço mal-intencionado em um computador de domínios. Este serviço pode gerar um ticket de serviço de Kerberos especialmente criado que permite que o invasor obtenha privilégios de nível de sistema na máquina local.
  
**Quais são os principais sistemas que correm riscos com a vulnerabilidade?**    
Principalmente as estações de trabalho e os servidores de terminal estão correndo risco. Os servidores correrão mais risco se usuários que não tiverem permissões administrativas suficientes puderem fazer logon nos servidores e executar programas. Entretanto, as práticas recomendadas não recomendam esse procedimento.
  
**O que a atualização faz?**    
Esta atualização elimina a vulnerabilidade, impedindo o uso de algoritmos fracos de hash tanto no Windows Kerberos quanto no Windows KDC.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Sim. Essa vulnerabilidade foi divulgada publicamente. Ela recebeu o número [CVE-2010-0043](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-0043) da lista Common Vulnerability and Exposure.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**    
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes e não testemunhou nenhum exemplo de publicação de código de verificação de conceito, quando este boletim de segurança foi lançado pela primeira vez.
  
Vulnerabilidade de falsificação de Kerberos - CVE-2011-0091  
-----------------------------------------------------------
  
Existe uma vulnerabilidade de falsificação nas implementações de Kerberos no Windows 7 e Windows Server 2008 R2. A vulnerabilidade existe porque é possível fazer downgrade da autenticação de Kerberos para usar o DES em vez dos padrões de criptografia padrão mais fortes incluídos no Windows 7 e Windows Server 2008 R2.
  
Para exibir esta vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2011-0091](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-0091).
  
#### Fatores atenuantes da vulnerabilidade de soma de verificação sem chave do Kerberos - CVE-2011-0091
  
A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:
  
-   A vulnerabilidade só afeta a autenticação de Kerberos.
  
#### Soluções alternativas para a vulnerabilidade de falsificação de Kerberos - CVE-2011-0091
  
A Microsoft não identificou nenhuma solução alternativa para esta vulnerabilidade.
  
#### Perguntas frequentes sobre a vulnerabilidade de falsificação de Kerberos - CVE-2011-0091
  
**Qual é o escopo da vulnerabilidade?**    
Essa é uma vulnerabilidade de falsificação. Esta vulnerabilidade pode permitir que um invasor personifique credenciais de usuários legítimos ou forje todo o tráfego de Kerberos em uma sessão comprometida.
  
**O que provoca a vulnerabilidade?**    
A vulnerabilidade é causada quando o Windows não impõe corretamente os padrões de criptografia padrão mais fortes incluídos no Windows 7 e Windows Server 2008 R2, fazendo com que seja possível que um invasor intermediário force um downgrade na comunicação de Kerberos entre um cliente e servidor a um padrão mais fraco de criptografia do que o negociado originalmente.
  
**O que é o Kerberos?**    
Kerberos é um protocolo usado mutuamente para autenticar usuários e serviços em uma rede aberta e sem proteção. Permite que os serviços identifiquem corretamente o usuário de uma permissão Kerberos, sem ter que autenticar o usuário no serviço. Isso é feito usando as chaves secretas compartilhadas.
  
O protocolo Kerberos usa chaves secretas compartilhadas para criptografar e assinar credenciais dos usuários. Um cliente é autenticado por um Centro de Distribuição de Chaves Kerberos (KDC). Depois dessa autenticação, o usuário pode solicitar uma permissão de serviço para acessar um serviço específico na rede. Essa permissão inclui a identidade autenticada criptografada do usuário.
  
**Para que um invasor pode usar a vulnerabilidade?**    
Um invasor pode baixar o padrão de criptografia padrão para DES. Em seguida, eles poderão ler e forjar todo o tráfego de Kerberos nessa sessão. Um invasor pode usar esse recurso para personificar o usuário que estava autenticando durante essa sessão de Kerberos.
  
**De que forma o invasor pode explorar a vulnerabilidade?**    
Em um cenário de ataque intermediário, um invasor teria que estar situado entre o cliente de destino e servidor para interceptar as comunicações e baixar o padrão de criptografia padrão para DES.
  
**Quais são os principais sistemas que correm riscos com a vulnerabilidade?**    
Os sistemas Windows 7 e Windows Server 2008 R2 que usam a autenticação de Kerberos estão vulneráveis a este ataque de downgrade.
  
**O que a atualização faz?**    
A atualização elimina a vulnerabilidade, impedindo que um invasor intermediário faça downgrade do padrão de criptografia para DE para a comunicação de Kerberos entre o cliente e servidor.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade por meio da divulgação coordenada de vulnerabilidades.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**    
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido usada publicamente para atacar clientes quando este boletim de segurança foi lançado pela primeira vez.

Quando você instala esta atualização de segurança, o instalador verifica se alguns dos arquivos que estão sendo atualizados no sistema foram anteriormente atualizados por um hotfix da Microsoft.
