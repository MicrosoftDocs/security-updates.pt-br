Microsoft Security Bulletin MS13-079 - Importante
=================================================

Vulnerabilidade no Active Directory pode permitir uma negação de serviço (2853587)
----------------------------------------------------------------------------------

Publicado: terça-feira, 10 de setembro de 2013

**Versão:** 1.0

### Informações Gerais

#### Sinopse

Esta atualização de segurança elimina uma vulnerabilidade relatada em particular no Active Directory. A vulnerabilidade pode permitir a negação de serviço se um invasor enviar uma consulta especialmente criada ao serviço Lightweight Directory Access Protocol (LDAP).

Esta atualização de segurança é classificada como Importante para o LDS do Active Directory (AD LDS) e Active Directory Services que afetam o Windows Vista, Windows Server 2008, Windows 7, Windows Server 2008 R2, Windows 8 e o Windows Server 2012.

A atualização de segurança elimina a vulnerabilidade, corrigindo a forma como o serviço de diretório LDAP controla consultas LDAP especialmente criadas. Para obter mais informações sobre a vulnerabilidade, consulte a subseção Perguntas frequentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A maioria dos clientes tem o recurso de atualizações automáticas habilitado e não precisará tomar nenhuma providência porque esta atualização de segurança será baixada e instalada automaticamente. Os clientes que não têm o recurso de atualizações automáticas habilitado precisarão verificar as atualizações e instalar esta atualização manualmente. Para obter informações sobre opções de configuração específicas em atualizações automáticas, consulte o [Artigo 294871 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Para instalações feitas por administradores ou empresas, ou por usuários finais que desejam instalar esta atualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a atualização assim que possível usando software de gerenciamento de atualização ou verificando se há atualizações no serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a seção **Orientação e ferramentas de detecção e implantação** mais adiante neste boletim.

Vulnerabilidade anônima DoS remota – CVE-2013-3868
--------------------------------------------------

Existe uma vulnerabilidade de DoS (negação de serviço) em implementações do Active Directory Services e AD LDS que podem fazer com que o serviço de diretório de LDAP pare de responder até que um administrador reinicie o serviço. A vulnerabilidade é causada quando o serviço de diretório LDAP não consegue controlar uma consulta especialmente criada.

Para exibir esta vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2013-3868](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2013-3868).

#### Fatores atenuantes

A Microsoft não identificou nenhuma atenuação para essa vulnerabilidade.

#### Soluções alternativas

A Microsoft não identificou nenhuma solução alternativa para esta vulnerabilidade.

#### Perguntas frequentes

**Qual é o escopo da vulnerabilidade?**  
Essa é uma vulnerabilidade de negação de serviço (site em inglês).

**O que provoca a vulnerabilidade?**  
A vulnerabilidade é causada quando o serviço de diretório LDAP não consegue controlar adequadamente uma consulta LDAP especialmente criada.

**O que é Active Directory?**  
O Active Directory é um componente do Windows que fornece autenticação central e serviços de autorização para sistemas Windows.

**O que é o Active Directory** Services? 
O Active Directory Services contém um conjunto extensível e escalonável de serviços que permitem a você eficientemente gerenciar identidades da empresa, credenciais, proteção de informações e configurações de sistema e de aplicativos. O Active Directory Services inclui Active Directory Certificate Services (AD CS), Active Directory Domain Services (AD DS), Active Directory Federation Services (AD FS), Active Directory Lightweight Directory Services (AD LDS) e Active Directory Rights Management Services (AD RMS).

**O que é o Active Directory Lightweight Directory Service (AD LDS)?**  
O Active Directory Lightweight Directory Services (AD LDS) é um modo independente do Active Directory que fornecem serviços de diretório dedicados para aplicativos. O AD LDS está disponível para Windows Vista, Windows Server 2008, Windows 7 e Windows Server 2008 R2. O AD LDS substitui o Active Directory Application Mode (ADAM), que está disponível para Windows XP e Windows Server 2003. Para obter mais informações sobre o AD LDS, consulte [Active Directory Lightweight Directory Service](http://technet.microsoft.com/library/cc754361).

**Para que um invasor pode usar a vulnerabilidade?**  
Um invasor que explorar de forma efetiva esta vulnerabilidade pode fazer com que o serviço de diretório LDAP deixe de responder até que um administrador reinicie o serviço.

**De que forma o invasor pode explorar a vulnerabilidade?**  
O invasor pode explorar esta vulnerabilidade, enviando uma consulta especialmente criada ao serviço LDAP.

**Quais são os principais sistemas que correm riscos com a vulnerabilidade?**  
Os servidores correm riscos maiores com essa vulnerabilidade.

**O que a atualização faz?**  
A atualização trata a vulnerabilidade corrigindo a forma como o serviço de diretório LDAP controla consultas LDAP especialmente criadas.

**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**  
Não. A Microsoft recebeu informações sobre esta vulnerabilidade por meio da divulgação coordenada de vulnerabilidades.

**Quando este boletim de segurança foi lançado, a Microsoft havia recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**  
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tinha sido usada publicamente para atacar clientes quando este boletim de segurança foi lançado pela primeira vez.

Orientação e ferramentas de detecção e implantação
--------------------------------------------------

Vários recursos estão disponíveis para ajudar administradores a implantar atualizações de segurança. 

-   O MBSA (Microsoft Baseline Security Analyzer) permite aos administradores pesquisar, em sistemas locais e remotos, atualizações de segurança ausentes e problemas de configuração de segurança comuns. 
-   O WSUS (Windows Server Update Services), SMS (Systems Management Server) e SCCM (System Center Configuration Manager) ajudam os administradores a distribuir as atualizações de segurança. 
-   Os componentes do Avaliador de compatibilidade com atualizações, incluídos no Kit de ferramentas de compatibilidade de aplicativos, auxilia a otimizar os testes e a validação das atualizações do Windows com relação aos aplicativos instalados. 

Para informações sobre estas e outras ferramentas que estão disponíveis, consulte [Ferramentas de segurança para profissionais de TI](http://technet.microsoft.com/security/cc297183).
