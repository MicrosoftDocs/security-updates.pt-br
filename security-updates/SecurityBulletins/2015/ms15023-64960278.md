Boletim de Segurança da Microsoft MS15-023 - Importante
=======================================================

Vulnerabilidades no driver de modo kernel podem permitir a elevação de privilégio (3034344)
-------------------------------------------------------------------------------------------

Publicado em: 10.03.15

**Versão:** 1.0

Sinopse
-------

<span id="sectionToggle0"></span>
Esta atualização de segurança elimina vulnerabilidades no Microsoft Windows. A mais severa destas vulnerabilidades poderá permitir a elevação de privilégio se um atacante se conectar a um sistema e executar um aplicativo especialmente criado para aumentar os privilégios. O atacante poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário.

Essa atualização de segurança foi classificada como Importante para todas as versões com suporte do Microsoft Windows. Para obter mais informações, consulte a seção **Software Afetado**.

A atualização de segurança aborda as vulnerabilidades modificando como o driver de modo kernel manipula objetos na memória e corrigindo como ele valida um do token de thread da chamada. Para obter mais informações sobre as vulnerabilidades, consulte a seção **Informações sobre vulnerabilidade**.

<span id="KBArticle"></span>
Para obter mais informações sobre essa atualização, consulte o [artigo 3034344 da Base de Dados de Conhecimento da Microsoft](https://support.microsoft.com/kb/3034344).

Informações sobre a vulnerabilidade
-----------------------------------

<span id="sectionToggle3"></span>
Vulnerabilidade de divulgação de memória de Kernel do Microsoft Windows – CVE-2015-0077
---------------------------------------------------------------------------------------

Existe uma vulnerabilidades de divulgação de informações no driver de modo kernel que pode permitir a divulgação de conteúdo de memória de kernel para um atacante. Essa vulnerabilidade é causada quando o driver de modo kernel falha ao inicializar os buffers de função de forma que remove os resultados de chamadas de função anteriores.

Um atacante pode usar esta divulgação de informações para obter informações sobre o sistema que poderia então ser combinado com outros ataques para comprometer o sistema. A vulnerabilidade de divulgação de informações por si só não permite a execução arbitrária de código. No entanto, um atacante poderá usá-la em conjunto com outra vulnerabilidade para ignorar recursos de segurança, como o ASLR (Address Space Layout Randomization). A atualização aborda esta vulnerabilidade modificando como o driver de modo kernel manipula os objetos na memória.

A Microsoft recebeu informações sobre esta vulnerabilidade por meio da divulgação coordenada de vulnerabilidades. Quando esse boletim de segurança foi emitido, a Microsoft ainda não tinha recebido informações que indicassem que a vulnerabilidade tinha sido usada de forma pública para atacar consumidores.

### Fatores atenuantes

A Microsoft não identificou quaisquer [fatores atenuantes](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) para essa vulnerabilidade.

### Soluções alternativas

A Microsoft não identificou nenhuma [solução alternativa](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) para esta vulnerabilidade.

Vulnerabilidade de elevação de privilégio do Win32k – CVE-2015-0078
-------------------------------------------------------------------

Existe uma vulnerabilidade de elevação de privilégio no driver de modo kernel do Windows que é causada quando o driver de modo kernel falha em validar adequadamente o token de thread da chamada.

Um atacante autenticado que explorou com êxito essa vulnerabilidade pode adquirir credenciais do administrador e usá-los para elevar privilégios. O atacante poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário. Para explorar a vulnerabilidade, o atacante precisa primeiro fazer logon no sistema. O atacante pode executar um aplicativo especialmente criado projetado para aumentar os privilégios. A atualização aborda a vulnerabilidade, corrigindo como o driver de modo kernel valida um token de thread da chamada.

A Microsoft recebeu informações sobre esta vulnerabilidade por meio da divulgação coordenada de vulnerabilidades. Quando esse boletim de segurança foi emitido, a Microsoft ainda não tinha recebido informações que indicassem que a vulnerabilidade tinha sido usada de forma pública para atacar consumidores.

### Soluções alternativas

Vulnerabilidade de divulgação de memória de Kernel do Microsoft Windows – CVE-2015-0095
---------------------------------------------------------------------------------------

Existe uma vulnerabilidades de divulgação de informações no driver de modo kernel que pode permitir a divulgação de conteúdo de memória de kernel para um atacante. Essa vulnerabilidade é causada quando o driver de modo kernel do Windows cancela a referência de um ponteiro NULL. A página NULA não é normalmente alocada ou mapeada, então se o kernel cancela a referência de um ponteiro NULL o resultado esperado é uma tela azul. No entanto, a tela azul pode ser evitada se a página NULA for mapeada e, como a página NULA reside no espaço de endereço de memória do modo do usuário, o atacante pode copias informações de kernel na página NULA e ler seu conteúdo. Este vetor de ataque não é viável quando o mapeamento da página NULA estiver desabilitado, que é o estado padrão no Windows 8 e sistemas operacionais mais recentes.

Um atacante pode usar esta divulgação de informações para obter informações sobre o sistema que poderia então ser combinado com outros ataques para comprometer o sistema.
