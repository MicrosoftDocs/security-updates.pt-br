Boletim de Segurança da Microsoft MS15-111 - Importante
=======================================================

Atualização de segurança do kernel do Windows para abordar a elevação de privilégio (3096447)
---------------------------------------------------------------------------------------------

Publicado em: 13/10/2015 | Atualizado em: 16/10/2015

**Versão:** 1.1

Sinopse
-------

Esta atualização de segurança resolve vulnerabilidades no Microsoft Windows. A mais severa das vulnerabilidades poderá permitir a elevação de privilégio se um atacante se conectar a um sistema afetado e executar um aplicativo especialmente criado.

**Observação** Clientes que estão usando soluções de atestado de relatório remoto devem rever os detalhe do CVE-2015-2552 discutidos neste boletim.

Essa atualização de segurança foi classificada como Importante para todas as versões com suporte do Microsoft Windows. Para obter mais informações, consulte a seção **Software afetado**.

A atualização de segurança aborda as vulnerabilidades corrigindo como o kernel do Windows lida com objetos na memória, corrigindo como o Windows lida com alguns cenários que envolvem junção e criação de ponto de montagem, e melhorando como o Windows analisa Dados de Configuração da Inicialização (BCD). Para obter mais informações sobre as vulnerabilidades, consulte a seção **Informações sobre vulnerabilidade**.

Para obter mais informações sobre essa atualização, consulte o [artigo 3096447 da Base de Dados de Conhecimento da Microsoft](https://support.microsoft.com/pt-br/kb/3096447).

Software afetado
----------------

As seguinte versões ou edições de software foram afetadas. As versões ou edições que não estão listadas já passaram de seu ciclo de vida do suporte ou não foram afetadas. Perguntas frequentes de atualização
-----------------------------------

**Estou executando o Windows 8.1, o Windows Server 2012 R2 ou o Windows RT 8.1. Por que a atualização 3088195 não está sendo oferecida?**  
Para receber a atualização de segurança 3088195 para Windows RT 8.1, Windows 8.1 e Windows Server 2012 R2, a atualização 2919355 deve ser instalada no sistema. Para obter informações adicionais, consulte o [artigo 2919355 da Base de Dados de Conhecimento Microsoft](https://support.microsoft.com/pt-br/kb/2919355).

Várias vulnerabilidades de elevação de privilégio de kernel do Windows
----------------------------------------------------------------------

Existem várias vulnerabilidades de elevação de privilégio na maneira como o kernel do Windows manipula objetos na memória. Um atacante que explorar com êxito esta vulnerabilidade poderá executar código arbitrário no modo do kernel. O atacante poderá instalar programas; exibir, alterar ou excluir dados; ou criar novas contas com direitos totais de usuário.

Para explorar as vulnerabilidades, o atacante precisa primeiro fazer logon no sistema. Em seguida, ele poderia executar um aplicativo especialmente criado capaz de explorar as vulnerabilidades e assumir o controle do sistema afetado. A atualização aborda a vulnerabilidade corrigindo o modo como o kernel do Windows manipula objetos na memória.

Vulnerabilidade de bypass de recurso de segurança de inicialização confiável – CVE-2015-2552  
--------------------------------------------------------------------------------------------
  
Existe uma vulnerabilidade de bypass de recurso de segurança quando o Windows não consegue aplicar corretamente a diretiva de [inicialização confiável](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) do Windows. Um atacante que tenha explorado com êxito essa vulnerabilidade poderá desabilitar as verificações de integridade de código, possibilitando que executáveis e drivers de teste assinado sejam carregados em um dispositivo de destino. Além disso, um atacante pode ignorar a validação da integridade de inicialização confiável para recursos de segurança do BitLocker e a criptografia no dispositivo.
  
Um atacante que tenha obtido privilégios de administrador ou que tenha acesso físico a um dispositivo de destino poderá explorar a vulnerabilidade aplicando uma configuração criada com má intenção aos Dados de Configuração da Inicialização (BCD). A atualização de segurança aborda a vulnerabilidade melhorando como o Windows analisa o BCD.
  
Essa vulnerabilidade foi divulgada de forma pública. Ela recebeu o número [CVE-2015-2552](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2015-2552) da lista Common Vulnerability and Exposure. No momento da publicação original deste boletim, a Microsoft não estava ciente de nenhum ataque tentando explorar essa vulnerabilidade.
  
### Fatores atenuantes
  
Os seguintes [fatores atenuantes](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) podem ser úteis no seu caso:
  
-   Para explorar a vulnerabilidade, um atacante precisa ter privilégios administrativos ou acesso físico ao dispositivo de destino.
  
### Soluções alternativas
  
As seguintes [soluções alternativas](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) podem ser úteis em seu caso:
  
-   **Configurar o BitLocker para usar o Trusted Platform Module (TPM) + proteção de PIN**
  
    Para habilitar o protetor de TPM e PIN, habilite a diretiva de grupo de proteção aprimorada da seguinte maneira:
  
    1.  Clique em **Iniciar**, em **Executar**, digite **gpedit.msc** e clique em **OK** para abrir o Editor de Política de Grupo Local.  
    2.  Em **Política de Computador Local**, navegue até Modelos Administrativos&gt; Componentes do Windows&gt; Criptografia de Unidade de Disco BitLocker &gt; Unidades de Sistema Operacional.  
    3.  No painel direito, clique duas vezes em **Exigir autenticação adicional na inicialização**.  
    4.  Na caixa de diálogo que aparece, clique em **Ativado**.  
    5.  Em **Opções de**, selecione **Exigir TPM** e **Exigir PIN de inicialização com TPM**.  
    6.  Clique em **Aplicar** e feche o Editor de Política de Grupo Local.  
    7.  Abra um prompt de comando com privilégios de administrador e digite o seguinte comando: ```

  
            manage-bde -protectors -add c: <OR OS volume letter>-tpmandpin
  
          
```
  
    8.  Insira um PIN quando solicitado.  
    9.  Reinicie o sistema.
  
     
  
    **Impacto da solução alternativa. **O usuário precisará digitar o PIN sempre que o computador for reiniciado.
  
    **Como desfazer a solução alternativa:**
  
    1.  Clique em **Iniciar**, em **Executar**, digite **gpedit.msc** e clique em **OK** para abrir o Editor de Política de Grupo Local.  
    2.  Em **Política de Computador Local**, navegue até Modelos Administrativos&gt; Componentes do Windows&gt; Criptografia de Unidade de Disco BitLocker &gt; Unidades de Sistema Operacional.  
    3.  No painel direito, clique duas vezes em **Exigir autenticação adicional na inicialização**.  
    4.  Na caixa de diálogo que aparece, clique em **Ativado**.  
    5.  Em **Opções de**, desmarque **Permitir TPM** e **Permitir PIN de inicialização com TPM**.  
    6.  Clique em **Aplicar** e feche o Editor de Política de Grupo Local.  
    7.  Abra um prompt de comando com privilégios de administrador e digite o seguinte comando: ```

  
            manage-bde -protectors -add c: -tpm
  
          
```
  
    8.  Reinicie o sistema.  
         
  
-   **Desative a validação de proteção de integridade de inicialização segura do BitLocker**
  
    Para desativar a validação de integridade de inicialização segura, você primeiro deve suspender temporariamente a proteção do BitLocker. Siga estas etapas na ordem apresentada:
  
    1.  Suspender a proteção do BitLocker  
        1.  Abra o **Painel de Controle**, clique em **Sistema e Segurança**e, em seguida, clique em **Criptografia de Unidade de Disco BitLocker**.  
        2.  Clique em **Suspender proteção** e, em seguida, em **Sim** quando a confirmação for solicitada.  
        3.  Clique em **Painel de Controle**.  
    2.  Desativar a inicialização segura  
        1.  Clique em **Iniciar**, em **Executar**, digite **gpedit.msc** e clique em **OK** para abrir o Editor de Política de Grupo Local.  
        2.  Em **Política de Computador Local**, navegue até Modelos Administrativos&gt; Componentes do Windows&gt; Criptografia de Unidade de Disco BitLocker &gt; Unidades de Sistema Operacional.  
        3.  Clique duas vezes em **Permitir Inicialização Segura para validação de integridade**.  
        4.  Na caixa de diálogo que aparece, clique em **Desativado**.  
        5.  Clique em **Aplicar** e feche o Editor de Política de Grupo Local.  
    3.  Retomar a proteção do BitLocker  
        1.  Abra o **Painel de Controle**, clique em **Sistema e Segurança**e, em seguida, clique em **Criptografia de Unidade de Disco BitLocker**.  
        2.  Clique em **Continuar Proteção**.  
        3.  Clique em **Painel de Controle**.
  
     
  
    **Impacto da solução alternativa.** Desativar a validação de integridade de inicialização segura pode fazer com que os sistemas entrem na recuperação do BitLocker com maior frequência quando você atualizar versões do firmware ou configurações de BCD.
  
    **Como desfazer a solução alternativa:**
  
    Para ativar a validação de integridade de inicialização segura, você primeiro deve suspender temporariamente a proteção do BitLocker. Siga estas etapas na ordem apresentada:
  
    1.  Suspender a proteção do BitLocker  
        1.  Abra o **Painel de Controle**, clique em **Sistema e Segurança**e, em seguida, clique em **Criptografia de Unidade de Disco BitLocker**.  
        2.  Clique em **Suspender proteção** e, em seguida, em **Sim** quando a confirmação for solicitada.  
        3.  Clique em **Painel de Controle**.  
    2.  Habilite a inicialização segura  
        1.  Clique em **Iniciar**, em **Executar**, digite **gpedit.msc** e clique em **OK** para abrir o Editor de Política de Grupo Local.  
        2.  Em **Política de Computador Local**, navegue até Modelos Administrativos&gt; Componentes do Windows&gt; Criptografia de Unidade de Disco BitLocker &gt; Unidades de Sistema Operacional.  
        3.  Clique duas vezes em **Permitir Inicialização Segura para validação de integridade**.

Vulnerabilidade de elevação de privilégio de ponto de montagem do Windows - CVE-2015-2553  
-----------------------------------------------------------------------------------------
  
Existe uma vulnerabilidade de elevação de privilégio quando o Windows valida de forma incorreta junções em determinados cenários onde pontos de montagem estão sendo criados. O atacante que explorar essa vulnerabilidade com êxito poderá executar código arbitrário no contexto de segurança do usuário atual, executando um aplicativo comprometido.
  
Para explorar essa vulnerabilidade, um atacante provavelmente teria que se aproveitar de outra vulnerabilidade que permita que ele execute código arbitrário em um aplicativo de área restrita. A atualização aborda a vulnerabilidade corrigindo como o Windows lida com certos cenários que envolvem a junção e a criação do ponto de montagem.
  
Essa vulnerabilidade foi divulgada de forma pública. Ela recebeu o número [CVE-2015-2553](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2015-2553) da lista Common Vulnerability and Exposure. No momento da publicação original deste boletim, a Microsoft não estava ciente de nenhum ataque tentando explorar essa vulnerabilidade.
  
### Fatores atenuantes
  
A Microsoft não identificou quaisquer [fatores atenuantes](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) para essa vulnerabilidade.
  
### Soluções alternativas
  
A Microsoft não identificou nenhuma [solução alternativa](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) para essa vulnerabilidade.
