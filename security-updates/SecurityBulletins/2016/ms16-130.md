Boletim de Segurança da Microsoft MS16-130 - Crítica
====================================================

Atualização de segurança para Microsoft Windows (3199172)
---------------------------------------------------------

Publicado em: 8 de novembro de 2016 | Atualizado em: 13 de dezembro de 2016

**Versão:** 2.0

Resumo executivo
----------------

<span id="sectionToggle0"></span>
Esta atualização de segurança elimina vulnerabilidades no Microsoft Windows. A mais grave dessas vulnerabilidades poderá permitir a execução remota de código se um invasor localmente autenticado executar um aplicativo especialmente criado.

Esta atualização de segurança foi classificada como Crítica para todas as versões compatíveis de Microsoft Windows. Para obter mais informações, consulte a seção **Softwares afetados e classificações de gravidade da vulnerabilidade**.

A atualização de segurança aborda vulnerabilidades ao:

-   Corrigir como o IME do Windows carrega DLLs.
-   Exigir o uso de caminhos UNC reforçados em tarefas agendadas.

Para obter mais informações sobre as vulnerabilidades, consulte a seção **Informações sobre a vulnerabilidade**.

<span id="KBArticle"></span>
Para obter mais informações sobre essa atualização, consulte o [artigo 3199172 da Base de Dados de Conhecimento da Microsoft](https://support.microsoft.com/pt-br/kb/3199172).

Softwares afetados e classificações de gravidade da vulnerabilidade
-------------------------------------------------------------------

<span id="sectionToggle1"></span>
As seguinte versões ou edições de software foram afetadas. As versões ou edições que não estão listadas já passaram de seu ciclo de vida do suporte ou não foram afetadas. Para determinar o ciclo de vida do suporte para sua versão ou edição de software, consulte [Ciclo de vida do suporte da Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

As classificações de gravidade indicadas para cada software afetado assumem o potencial máximo do impacto da vulnerabilidade. Para obter informações sobre a probabilidade de exploração da vulnerabilidade em relação à sua classificação de gravidade e ao impacto à segurança, dentro de 30 dias a partir do lançamento deste boletim de segurança, consulte o Índice de exploração no [Resumo de boletins de novembro](https://technet.microsoft.com/pt-br/library/security/ms16-nov).

Vulnerabilidade de elevação de privilégios do Windows IME - CVE-2016-7221
-------------------------------------------------------------------------

Existe uma vulnerabilidade de elevação de privilégio no Windows quando o Windows IME manipula o carregamento de DLLs indevidamente. Não há impacto quando um IME não está presente. 

Para explorar essa vulnerabilidade, um invasor localmente autenticado pode executar um aplicativo especialmente criado.

A atualização resolve essa vulnerabilidade, corrigindo a maneira como o Windows IME carrega DLLs.

Vulnerabilidade de elevação de privilégio do Agendador de Tarefas - CVE-2016-7222  
---------------------------------------------------------------------------------
  
Existe uma vulnerabilidade de elevação de privilégios no Agendador de Tarefas quando um usuário cria uma tarefa que usa caminhos UNC. Um invasor que conseguir explorar essa vulnerabilidade poderá executar código arbitrário com privilégios de sistema elevados.
  
Para explorar a vulnerabilidade, um invasor localmente autenticado pode usar o Agendador de Tarefas do Windows para agendar uma nova tarefa com um caminho UNC especialmente criado.
  
Esta atualização de segurança resolve a vulnerabilidade, corrigindo como o Agendador de Tarefas valida caminhos UNC especialmente criados.

Vulnerabilidade de execução remota de código do Windows - CVE-2016-7212  
-----------------------------------------------------------------------
  
Existe uma vulnerabilidade de execução remota de código quando a funcionalidade de carregamento de arquivos de imagem do Windows não manipula arquivos de imagem malformados corretamente. Um invasor que conseguir explorar essa vulnerabilidade poderá executar código arbitrário.
  
Para explorar a vulnerabilidade, o invasor precisa convencer um usuário a carregar um arquivo de imagem malformado de uma página da Web ou mensagem de email.
  
A atualização resolve essa vulnerabilidade, ajudando a garantir que o conteúdo de arquivos de imagem seja devidamente validado.
