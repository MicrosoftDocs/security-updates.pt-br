Security Bulletin

Microsoft Security Bulletin MS08-062 - Importante
=================================================

Vulnerabilidade no Windows Internet Printing Service pode permitir a execução remota de código (953155)
-------------------------------------------------------------------------------------------------------

Publicado: terça-feira, 14 de outubro de 2008 | Atualizado: quarta-feira, 29 de outubro de 2008

**Versão:** 2.2

### Informações Gerais

#### Sinopse

Esta atualização elimina uma vulnerabilidade no Windows Internet Printing Service, relatada em particular, que pode permitir a execução remota de código. O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado. Um invasor poderá instalar programas, visualizar, alterar ou excluir dados, ou ainda criar novas contas.

Esta atualização é classificada como Importante para todas as edições com suporte do Microsoft Windows 2000, Windows XP, Windows Server 2003 e Windows Server 2008. Para obter mais informações, consulte a subseção **Software afetado e não afetado** nesta seção.

Esta atualização de segurança elimina esta vulnerabilidade, mudando a maneira como a memória é alocada no serviço do Protocolo IPP (Internet Printing Protocol). Para obter mais informações sobre a vulnerabilidade, consulte a subseção Perguntas freqüentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A Microsoft recomenda que os clientes instalem a atualização o mais rápido possível.

**Problemas conhecidos.** Nenhuma

Perguntas freqüentes relacionadas a esta atualização de segurança
-----------------------------------------------------------------

<span></span>
**Por que este boletim foi revisado em 29 de outubro de 2008?**  
A Microsoft revisou este boletim para notificar aos clientes de que os pacotes de atualização do Windows Server 2008 para sistemas baseados no Itanium e todas as edições com suporte do Windows Vista agora foram disponibilizados no Windows Update, Microsoft Update, Windows Software Update Services (WSUS), Systems Management Server e System Center Configuration Manager. Quando este boletim foi lançado pela primeira vez, as atualizações para essas plataformas só estavam disponíveis no Centro de Download da Microsoft.

**Por que este boletim foi revisado em 16 de outubro de 2008?**
A Microsoft revisou este boletim para esclarecer que o serviço Windows Internet Printing é executado no contexto do serviço de Spooler, que é executado com privilégios de sistema. Um invasor que explorar com êxito a vulnerabilidade descrita neste boletim de segurança poderá executar código arbitrário com privilégios de sistema e obter controle total sobre o sistema afetado.

**Por que este boletim foi revisado em 15 de outubro de 2008?**
A Microsoft revisou este boletim para remover a classificação de gravidade para o Windows Server 2008 para sistemas baseados no Itanium. A Microsoft também atualizou este boletim para esclarecer que a atualização para Windows Server 2008 para sistemas baseados no Itanium está disponível através do Centro de Download da Microsoft e será lançada pelos outros canais de distribuição o quanto antes.

**Por que o Windows Server 2008 para sistemas baseados no Itanium e o Windows** **Vista** **estão listados na tabela de Softwares afetados se não têm nenhuma classificação de gravidade?**
O Windows Server 2008 para sistemas baseados no Itanium e todas as edições com suporte do Windows Vista contêm o componente de spooler de impressão compartilhada atualizado por esta atualização de segurança. No entanto, como a Impressão via Internet não é suportada no Windows Server 2008 para sistemas baseados no Itanium nem no Windows Vista, o vetor de exploração abordado neste boletim de segurança não é exposto. Como uma medida completa de defesa para proteger contra possíveis novos vetores identificados no futuro, aos usuários do Windows Server 2008 para sistemas baseados no Itanium e do Windows Vista é recomendado aplicar a atualização de segurança.

**Estou usando uma das plataformas que estão listadas na tabela de Softwares afetados. Por que não estão me oferecendo a atualização?**
A atualização somente será oferecida a sistemas em que o componente afetado esteja instalado. Os sistemas Microsoft Windows 2000 e Windows XP que não têm o IIS (Microsoft Internet Information Services) instalado não são afetados e não receberão esta atualização. Os sistemas Windows Server 2003 e Windows Server 2008 que não têm IIS nem impressão de Internet instalados não são afetados e não receberão esta atualização.

Vulnerabilidade no serviço IPP na saturação de número inteiro - CVE-2008-1446  
-----------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código em sistemas IIS (Microsoft Internet Information Services) com o serviço Windows Internet Printing habilitado. Esse problema pode permitir que um invasor autenticado remoto execute código arbitrário em um sistema afetado.
  
Para exibir essa vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2008-1446](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1446).
  
#### Fatores atenuantes para a Vulnerabilidade de serviço IPP na saturação do número inteiro - CVE-2008-1446
  
A atenuação refere-se a uma configuração, configuração comum ou prática recomendada geral, existindo num estado padrão, que possa reduzir a gravidade de exploração de uma vulnerabilidade. Os fatores de atenuação a seguir podem ser úteis em sua situação:
  
-   O Protocolo IPP (Internet Printing Protocol) está habilitado por padrão no IIS para Microsoft Windows 2000 e Windows XP. No entanto, as permissões de arquivo padrão entre todas as plataformas são configuradas de forma que o IPP só possa ser acessado por usuários no grupo "Autenticado". Por causa desta restrição, os usuários não autenticados por padrão não terão as permissões de pasta necessárias para explorar esta vulnerabilidade remotamente.  
-   As instalações do IIS no Windows Server 2003 não estão vulneráveis em sua configuração padrão. Somente as instalações do Windows Server 2003 que ativaram o filtro ISAPI do Protocolo IPP no ISS estão vulneráveis. Tanto os componentes Internet Printing quanto Active Server Pages são necessários para habilitar o gerenciamento de impressão com base na Web e a impressão de Internet. Se o componente Active Server Pages for definido como "Permitido" e o componente Internet Printing for definido como "Proibido", somente o gerenciamento de impressora com base na Web estará habilitado. Os usuários não podem se conectar a nenhuma impressora compartilhada por meio do Internet Printing.  
-   Por padrão, as instalações do IIS no Windows Server 2008 não estão vulneráveis. Um computador que executa o Windows Server 2008 pode agir como um computador cliente que usa o Internet Printing. No entanto, os usuários do computador que fazem solicitações de impressão devem ser autenticados pelo servidor de impressão antes de poderem usar qualquer uma das impressoras conectadas a esse servidor. Para habilitar um computador que executa o Windows Server 2008 a agir como um servidor que oferece suporte a Internet Printing, você deverá instalar a função servidor Web (IIS), a função Serviços de impressão e o serviço de função Impressão de Internet na função Serviços de impressão.  
-   A exploração desta vulnerabilidade requer que o servidor IIS comunique a saída para um computador mal-intencionado na porta 445. Os administradores que bloqueiam as conexões de SMB de saída do seu servidor IIS para a Internet podem evitar a exploração desta vulnerabilidade a partir de sistemas não confiáveis.
  
#### Soluções alternativas para a Vulnerabilidade de serviço IPP na saturação do número inteiro - CVE-2008-1446
  
A solução alternativa refere-se a uma configuração ou alteração de configuração que não corrige a vulnerabilidade subjacente mas que ajudaria a bloquear vetores de ataque conhecidos antes de aplicar a atualização. A Microsoft testou as soluções alternativas e estados a seguir como parte do cenário que implica se uma solução alternativa reduziria funcionalidade:
  
-   **Desabilitar o serviço IPP**
  
    Para desabilitar o serviço IPP no IIS 5.0 e IIS 5.1
  
    1.  Abra o **Gerenciador de Serviços de Internet**.  
    2.  Clique com o botão direito do mouse no **Site padrão** e depois em **Propriedades**.  
    3.  Vá até a guia de **Diretório Base** e selecione **Configuração**.  
    4.  Role para encontrar a extensão **.printer** e o caminho correspondente para **msw3prt.dll**.  
    5.  Clique em **Remover**.  
    6.  Clique em **OK**.  
    7.  Reinicie o Serviço IIS.
  
    **Observação** A Impressão de Internet pode ser configurada pela diretiva de grupo bem como pelo Gerenciador de Serviços de Internet. Se houver um conflito entre as configurações de diretiva de grupo e as configurações do Gerenciador de Serviços de Internet, as configurações de diretiva de grupo terão precedência. Se você remover a Impressão de Internet via Gerenciador de Serviços de Internet, certifique-se de verificar se ele não será reativado pelas diretivas de grupo de domínio ou local. (A diretiva de grupo padrão nem habilita nem desabilita a Impressão de Internet). No snap-in de Diretiva de Grupo MMC, marque Configuração do Computador | Modelos Administrativos | Impressão |Impressão baseada na Web.
  
    Para desabilitar o serviço IPP no IIS 6.0
  
    1.  Inicie o **Gerenciador de IIS** ou o **snap-in de IIS**.  
    2.  Expanda **\*server\_name**, sendo que server\_name é o nome do servidor.  
    3.  Clique em **Extensões de Serviço da Web**.  
    4.  No painel direito, clique em **Impressão de Internet** e depois em **Proibido**.  
    5.  Saia do Gerenciador de IIS.
  
    Para desabilitar o Serviço IPP no IIS 7.0 no Windows Server 2008
  
    1.  Se o **Gerenciador de Servidores** não estiver sendo executado, clique em **Iniciar**, em **Ferramentas Administrativas** e depois em **Gerenciador de Servidores**. (Se a caixa de diálogo **Controle da conta do usuário** for exibida, confirme se a ação que é exibida é a desejada e clique em **Continuar**.)  
    2.  Expanda o nó Funções e clique com o botão direito do mouse em **Serviços de Impressão** e selecione **Remover Serviços de Função**.  
    3.  O Assistente para Remover Serviços de Função será exibido. Se a opção **Impressão de Internet** estiver marcada, desmarque a caixa de seleção, clique em **Avançar** e depois em **Remover** para concluir o Assistente.
  
    **Impacto da solução alternativa.** Os usuários não poderão se conectar a nenhuma impressora compartilhada pela Impressão de Internet.
  
    **Como desfazer a solução alternativa.**
  
    Para habilitar o serviço IPP no IIS 5.0 e IIS 5.1
  
    1.  Abra o **Gerenciador de Serviços de Internet**.  
    2.  Clique com o botão direito do mouse no **Site padrão** e depois em **Propriedades**.  
    3.  Vá até a guia de **Diretório Base** e selecione **Configuração**.  
    4.  Role para encontrar a extensão **.printer** e o caminho correspondente para **msw3prt.dll**.  
    5.  Clique em **Adicionar**.  
    6.  No campo marcado como **Executável**, digite:  
        %systemroot%\\system32\\msw3prt.dll  
    7.  No campo marcado como **Extensão**, digite:  
        .printer  
    8.  No campo marcado como **Limitar a**, digite:  
        GET,POST  
    9.  Assim que isso for concluído, reinicie o serviço IIS.
  
    Para habilitar o serviço IPP no IIS 6.0
  
    1.  Inicie o **Gerenciador de IIS** ou o **snap-in de IIS**.  
    2.  Expanda **\*server\_name**, sendo que server\_name é o nome do servidor.  
    3.  Clique em **Extensões de Serviço da Web**.  
    4.  No painel direito, clique em **Impressão de Internet** e depois em **Permitir**.  
    5.  Saia do Gerenciador de IIS.
  
    Para habilitar o serviço IPP no IIS 7.0
  
    1.  Se o **Gerenciador de Servidores** não estiver sendo executado, clique em **Iniciar**, em **Ferramentas Administrativas** e depois em **Gerenciador de Servidores**. (Se a caixa de diálogo **Controle da conta do usuário** for exibida, confirme se a ação que é exibida é a desejada e clique em **Continuar**.)  
    2.  Expanda o nó Funções e clique com o botão direito do mouse em **Serviços de Impressão** e selecione **Adicionar Serviços de Função**.  
    3.  O Assistente para Adicionar Serviços de Função será exibido. Marque a opção **Impressão de Internet**, clique em **Avançar** e depois em **Instalar** para concluir o Assistente.
  
-   **Executar o ISS Lockdown Tool 2.1**
  
    No Microsoft Windows 2000 que executa o IIS 5.0 ou no Windows XP que executa o IIS 5.1, o ISS Lockdown Wizard versão 2.1 pode ser usado para desabilitar os recursos desnecessários, inclusive a Impressão de Internet. Para executar este assistente, conclua as seguintes ações:
  
    1.  Baixe a IIS Lockdown Tool 2.1 [daqui](http://www.microsoft.com/downloads/details.aspx?familyid=dde9efc0-bb30-47eb-9a61-fd755d23cdec&displaylang=en).  
    2.  Execute uma das seguintes ações:  
        Para iniciar a instalação imediatamente, clique em **Abrir** ou **Executar este programa do seu local atual**.  
        Para copiar o download para seu computador para instalação posterior, clique em **Salvar** ou **Salvar este programa no disco**.
  
    **Impacto da solução alternativa.**
  
    Os usuários não poderão se conectar a nenhuma impressora compartilhada pela Impressão de Internet. Adicionalmente, os Usuários da ferramenta IIS Lockdown não poderão se conectar a nenhuma impressora compartilhada pela Impressão de Internet. Para obter uma lista completa das alterações feitas pela ferramenta ISS Lockdown, consulte o seguinte [artigo do MSDN](http://msdn.microsoft.com/en-us/library/aa302365.aspx).
  
    **Como desfazer a solução alternativa.**
  
    Para desfazer as alterações feitas pelo IIS Lockdown, execute o **IISlockd.exe** uma segunda vez e escolha desfazer as alterações. A operação para desfazer restaura as configurações do sistema que estavam em vigor imediatamente antes de você executar o IISLockdown anteriormente. Esses detalhes estão contidos no arquivo de log \\WINNT\\System32\\inetsrv\\0blt-log. Portanto, é importante que você teste o sistema imediatamente após executar o IISLockdown. Se uma ação para desfazer for necessária, execute-a imediatamente.
  
    **Observação** O filtro ISAPI do URLscan instalado como parte do IIS Lockdown não é removido como parte do processo para desfazer. Você pode remover o URLscan manualmente, usando a guia de filtros ISAPI no nível do servidor no Gerenciador de Serviços de Internet.
  
#### Perguntas freqüentes sobre a Vulnerabilidade de serviço IPP na saturação do número inteiro - CVE-2008-1446
  
**Qual é o escopo da vulnerabilidade?**    
Essa é uma vulnerabilidade de execução remota de código. O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado. Um invasor poderá instalar programas, visualizar, alterar ou excluir dados, ou ainda criar novas contas.
  
**O que provoca a vulnerabilidade?**    
A função IPP (Internet Printing Protocol) é implementada como uma extensão ISAPI para IIS. Existe uma vulnerabilidade remota autenticada na implementação do IPP nos servidores do Windows que estão executando o IIS. Existe uma saturação de número inteiro na implementação de IPP que pode permitir que um invasor autenticado execute a execução remota de código em servidores afetados de IIS.
  
**O que é ISAPI?**  
O ISAPI (Internet Services Application Programming Interface) é uma tecnologia que permite que os desenvolvedores da Web estendam a funcionalidade de seus servidores da Web, escrevendo código personalizado que fornece novos serviços para um servidor Web. O código personalizado pode ser implementado em um filtro ISAPI se a funcionalidade nova fornecer um serviço de baixo nível ou uma extensão ISAPI se a funcionalidade nova fornecer um serviço de alto nível. Nesse caso, o código afetado será uma extensão ISAPI.
  
**O que é a extensão ISAPI em questão aqui?**  
A extensão ISAPI afetada é aquela que implementa o Protocolo IPP, um padrão de mercado definido nas RFCs 2910 e 2911. O IPP fornece uma maneira de solicitar serviços de impressão e conhecer o status de trabalhos de impressão por meio da Internet via HTTP. Por exemplo, usando IPP, um usuário em trânsito pode enviar um trabalho de impressão pela Internet para ser impresso em uma impressora em sua rede corporativa. O usuário também pode saber se a solicitação de impressão foi concluída sem erros.
  
O Microsoft Windows 2000 introduziu suporte nativo para a Impressão de Internet. A implementação do Microsoft Windows 2000 permite que os usuários imprimam diretamente para um URL e vejam informações sobre trabalhos de impressão usando seus navegadores. O suporte para Impressão de Internet está habilitado por padrão nas versões do Microsoft Windows 2000 Server. Ele também pode ser instalado como um componente opcional no Microsoft Windows 2000 Professional, Windows XP, Windows Server 2003 e Windows Server 2008.
  
**Para que um invasor pode usar a vulnerabilidade?**    
O invasor que explorar com êxito esta vulnerabilidade poderá executar código arbitrário no sistema de um usuário com privilégios de sistema. O invasor que explorar com êxito essa vulnerabilidade poderá assumir o controle total de um sistema afetado. Um invasor poderá instalar programas, visualizar, alterar ou excluir dados, ou ainda criar novas contas.
  
**De que forma o invasor pode explorar a vulnerabilidade?**    
Um invasor pode enviar uma solicitação POST HTTP especialmente criada a um servidor Web vulnerável que faria com que o servidor se conectasse a uma máquina controlada pelo invasor agindo como uma impressora que usa IPP. O invasor então usaria respostas especialmente criadas para executar a execução remota de código no servidor de destino.
  
**Quais são os sistemas que mais correm riscos com a vulnerabilidade?**    
As estações de trabalhos e os servidores que executam o IIS com o serviço Protocolo IPP habilitado são os que mais correm risco com esta vulnerabilidade.
  
**O que a atualização faz?**    
Esta atualização de segurança elimina esta vulnerabilidade, validando os dados recebidos de máquinas remotas no serviço Protocolo IPP.
  
**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre essa vulnerabilidade por meio de divulgação responsável. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tenha sido divulgada publicamente quando este boletim de segurança foi lançado pela primeira vez.
  
**Quando este boletim de segurança foi lançado, a Microsoft tinha recebido algum relatório informando que essa vulnerabilidade estava sendo explorada?**    
Sim, a Microsoft está ciente dos ataques direcionados que tentam explorar a vulnerabilidade. No entanto, a Microsoft não tinha visto nenhum exemplo de código de prova de conceito publicado quando este boletim de segurança foi originalmente publicado.

**Verificando a aplicação da atualização**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma atualização de segurança foi aplicada em um sistema afetado, é possível utilizar a ferramenta MBSA (Microsoft Baseline Security Analyzer). Consulte a seção Orientação e ferramentas de detecção e implantação neste boletim para obter mais informações.

-   **Verificação de Versão do Arquivo**

    Como existem várias versões do Microsoft Windows, as etapas a seguir podem ser diferentes no seu sistema. Se isso acontecer, consulte a documentação do produto para completar estas etapas.

    1.  Clique em **Iniciar** e, em seguida, clique em **Pesquisar**.
    2.  No painel **Resultados da Pesquisa**, clique em **Todos os arquivos e pastas**, em **Search Companion**.
    3.  Na caixa **Todo ou parte do nome do arquivo**, digite um nome de arquivo da tabela de informações de arquivo correta e clique em **Pesquisar**.
    4.  Na lista de arquivos, clique com o botão direito do mouse em um nome de arquivo da tabela de informações de arquivo correta e clique em **Propriedades**.
        **Observação** Dependendo da edição do sistema operacional, ou dos programas que são instalados em seu sistema, alguns arquivos que estão listados na tabela de informações de arquivo podem não estar instalados.
    5.  Na guia **Versão**, determine a versão do arquivo instalado no sistema, comparando-o com a versão documentada na tabela de informações de arquivos correta.
        **Observação** Atributos que não sejam a versão do arquivo podem ser alterados durante a instalação. A comparação de outros atributos de arquivo com as informações na tabela não é um método suportado para verificar se a atualização foi aplicada. Além disso, em alguns casos, arquivos podem ser renomeados durante a instalação. Se as informações sobre o arquivo ou a versão não estiverem presentes, use um dos outros métodos disponíveis para verificar a instalação da atualização.

-   **Verificação da Chave do Registro**

    Você também pode ser capaz de verificar os arquivos que esta atualização de segurança instalou examinando as chaves do Registro na **Tabela de referência** nesta seção.

    Essas chaves do Registro podem não conter uma lista completa dos arquivos instalados. Além disso, talvez essas chaves do Registro não sejam criadas corretamente se um administrador ou um OEM integrar ou adicionar a atualização de segurança aos arquivos fonte da instalação do Windows.
