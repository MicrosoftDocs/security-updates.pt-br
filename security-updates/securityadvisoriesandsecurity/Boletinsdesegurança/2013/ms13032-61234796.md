Security Bulletin

Microsoft Security Bulletin MS13-032 - Importante
=================================================

Vulnerabilidade no Active Directory pode permitir a negação de serviço (2830914)
--------------------------------------------------------------------------------

Publicado: terça-feira, 9 de abril de 2013

**Versão:** 1.0

### Informações Gerais

#### Sinopse

Esta atualização de segurança elimina uma vulnerabilidade relatada em particular no Active Directory. A vulnerabilidade pode permitir a negação de serviço se um invasor enviar uma consulta especialmente criada ao serviço Lightweight Directory Access Protocol (LDAP).

Esta atualização de segurança é classificada Importante para Active Directory, Active Directory Application Mode (ADAM), Active Directory Lightweight Directory Service (AD LDS) e Active Directory Services em servidores do Microsoft Windows (excluindo sistemas baseados em Itanium), e classificada como Baixa em clientes do Microsoft Windows.

A atualização de segurança trata a vulnerabilidade corrigindo como o serviço LDAP controla consultas especialmente criadas. Para obter mais informações sobre a vulnerabilidade, consulte a subseção Perguntas frequentes relacionada à entrada da vulnerabilidade específica presente na próxima seção, **Informações sobre a vulnerabilidade**.

**Recomendação.** A maioria dos clientes tem o recurso de atualizações automáticas habilitado e não precisará tomar nenhuma providência porque esta atualização de segurança será baixada e instalada automaticamente. Os clientes que não têm o recurso de atualizações automáticas habilitado precisarão verificar as atualizações e instalar esta atualização manualmente. Para obter informações sobre opções de configuração específicas em atualizações automáticas, consulte o [Artigo 294871 (em inglês) da Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Para instalações feitas por administradores ou empresas, ou por usuários finais que desejam instalar esta atualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a atualização assim que possível usando software de gerenciamento de atualização ou verificando se há atualizações no serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a seção **Orientação e ferramentas de detecção e implantação** mais adiante neste boletim.

Vulnerabilidade de consumo de memória - CVE-2013-1282
-----------------------------------------------------

<span></span>
Existe uma vulnerabilidade de negação de serviço em implementações do Active Directory que pode fazer com que o serviço pare de responder. A vulnerabilidade é causada quando o serviço LDAP não consegue controlar uma consulta especialmente criada.

Para exibir esta vulnerabilidade como uma entrada padrão na lista Common Vulnerabilities and Exposures, consulte [CVE-2013-1282](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2013-1282).

#### Fatores atenuantes

O invasor deve ter credenciais de logon válidas para explorar esta vulnerabilidade. Essa vulnerabilidade não pode ser explorada remotamente por usuários anônimos. No entanto, o componente afetado está disponível remotamente a usuários que têm contas de usuário normais. Em determinadas configurações, usuários anônimos podem autenticar como a conta de Convidado.

#### Soluções alternativas

A Microsoft não identificou nenhuma solução alternativa para esta vulnerabilidade.

#### Perguntas frequentes

**Qual é o escopo da vulnerabilidade?** 
Essa é uma vulnerabilidade de negação de serviço (site em inglês).

**O que provoca a vulnerabilidade?** 
A vulnerabilidade é causada quando o serviço LDAP não consegue controlar adequadamente uma consulta especialmente criada.

**O que é Active Directory?** 
O Active Directory é um componente do Windows que fornece autenticação central e serviços de autorização para sistemas Windows.

**O que é o Active Directory** Services? 
O Active Directory Services contém um conjunto extensível e escalonável de serviços que permitem a você eficientemente gerenciar identidades da empresa, credenciais, proteção de informações e configurações de sistema e de aplicativos. O Active Directory Services inclui Active Directory Certificate Services (AD CS), Active Directory Domain Services (AD DS), Active Directory Federation Services (AD FS), Active Directory Lightweight Directory Services (AD LDS) e Active Directory Rights Management Services (AD RMS).

**O que é o Active Directory Lightweight Directory Service (AD LDS)?  **
O Active Directory Lightweight Directory Services (AD LDS) é um modo independente do Active Directory que fornece serviços de diretório dedicados para aplicativos. O AD LDS está disponível para Windows Vista, Windows Server 2008, Windows 7 e Windows Server 2008 R2. O AD LDS substitui o Active Directory Application Mode (ADAM), que está disponível para Windows XP e Windows Server 2003. Para obter mais informações sobre o AD LDS, consulte o artigo do TechNet, [Active Directory Lightweight Directory Services](http://technet.microsoft.com/en-us/library/cc731868(ws.10).aspx).

**O que é o Active Directory Application Mode?  **
ADAM (Active Directory Application Mode) é um serviço de diretório LDAP (Lightweight Directory Access Protocol) que é executado como um serviço de usuário para Windows XP e Windows Server 2003, em vez de um serviço do sistema. Para obter mais informações, consulte o artigo do TechNet, [Introdução ao ADAM](http://technet.microsoft.com/en-us/library/cc780837(ws.10).aspx).

**O que é o LDAP (Lightweight Directory Access Protocol)?  **
O LDAP (Lightweight Directory Access Protocol) é um protocolo de serviço de diretório que é executado diretamente sobre a pilha de TCP/IP. O modelo de informação (tanto para dados como para namespaces) do LDAP é semelhante ao do serviço de diretório OSI X.500, mas com menos recursos e requisitos de recursos mais baixos que os do X.500. Ao contrário da maioria dos outros protocolos da Internet, o LDAP tem uma API associada que simplifica a criação de aplicativos de serviço de diretório da Internet. A API do LDAP aplica-se a aplicativos de navegador e gerenciamento de diretórios que não têm o suporte ao serviço de diretório como sua função principal. O LDAP não pode criar diretórios nem especificar como funciona um serviço de diretório.

**Para que um invasor pode usar a vulnerabilidade?** 
O invasor que conseguir explorar esta vulnerabilidade pode fazer com que o serviço LDAP pare de responder.

**De que forma o invasor pode explorar a vulnerabilidade?** 
O invasor pode explorar esta vulnerabilidade, enviando uma consulta especialmente criada ao serviço LDAP.

**Quais são os principais sistemas que correm riscos com a vulnerabilidade?** 
Os servidores correm riscos maiores com essa vulnerabilidade.

**O que a atualização faz?** 
A atualização trata a vulnerabilidade corrigindo como o serviço LDAP controla consultas especialmente criadas.

**Quando esse boletim de segurança foi lançado, essa vulnerabilidade já tinha sido divulgada publicamente?** 
Não. A Microsoft recebeu informações sobre esta vulnerabilidade por meio da divulgação coordenada de vulnerabilidades.

**Quando este boletim de segurança foi lançado, a Microsoft havia recebido algum relatório informando que essa vulnerabilidade estava sendo** **explorada?** 
Não. A Microsoft não recebeu nenhuma informação indicando que essa vulnerabilidade tinha sido usada publicamente para atacar clientes quando este boletim de segurança foi lançado pela primeira vez.
