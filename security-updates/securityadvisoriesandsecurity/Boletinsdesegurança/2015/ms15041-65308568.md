Boletim de Segurança da Microsoft MS15-041 - Importante
=======================================================

Vulnerabilidade no .Net Framework pode permitir a divulgação não autorizada de informações (3048010)
----------------------------------------------------------------------------------------------------

Publicado em: 14.04.15 | Atualizado em: 12.05.15

**Versão:** 2.0

Resumo executivo
----------------

<span id="sectionToggle0"></span>
Esta atualização de segurança resolve uma vulnerabilidade no Microsoft .NET Framework. A vulnerabilidade pode permitir divulgação não autorizada de informações se um atacante enviar uma solicitação da web especialmente criada para um servidor afetado que tem mensagens de erro personalizadas. Um atacante que explorar com êxito a vulnerabilidade poderia visualizar partes do arquivo de configuração da web, o que pode expor informações confidenciais.

Esta atualização de segurança é classificada como Importante para Microsoft .NET Framework 1.1 Service Pack 1, Microsoft .NET Framework 2.0 Service Pack 2, Microsoft .NET Framework 3.5, Microsoft .NET Framework 3.5.1, Microsoft .NET Framework 4, Microsoft .NET Framework 4.5, Microsoft .NET Framework 4.5.1 e Microsoft .NET Framework 4.5.2 em lançamentos do Microsoft Windows afetados. Para obter mais informações, consulte a seção **Software Afetado**.

A atualização de segurança aborda a vulnerabilidade ao remover os detalhes do conteúdo do arquivo das mensagens de erro que estavam facilitando a divulgação não autorizada de informações. Para obter mais informações, consulte a subseção **Perguntas Frequentes (FAQ)** relacionada à essa vulnerabilidade específica.

<span id="KBArticle"></span>
Para obter mais informações sobre essa atualização, consulte o [artigo 3048010 da Base de Dados de Conhecimento da Microsoft](https://support.microsoft.com/pt-br/kb/3048010).

Perguntas frequentes de atualização
-----------------------------------

<span id="sectionToggle2"></span>
**Como determino que a versão do Microsoft .NET Framework está instalada?**
Você pode instalar e executar várias versões do .Net Framework em um sistema, além de poder instalar as versões em qualquer ordem. Para obter mais informações, consulte o [Artigo 318785 (em inglês) da Microsoft Knowledge Base](https://support.microsoft.com/pt-br/kb/318785).****

**Qual é a diferença entre o .NET Framework 4 e o .NET Framework 4 Client Profile?
**Os pacotes redistribuíveis do .NET Framework versão 4 estão disponíveis em dois perfis: .NET Framework 4 e .NET Framework 4 Client Profile. O .NET Framework 4 Client Profile é um subconjunto do perfil do .NET Framework 4 que é otimizado para aplicativos clientes. Ele fornece funcionalidade para a maioria dos aplicativos clientes, incluindo recursos do Windows Presentation Foundation (WPF), Windows Forms, Windows Communication Foundation (WCF) e ClickOnce. Isso possibilita a implantação mais rápida e um pacote de instalação menor para aplicativos que têm como alvo o .NET Framework 4 Client Profile. Para obter mais informações, consulte o artigo de MSDN, [.NET Framework Client Profile](http://msdn.microsoft.com/pt-br/library/cc656912).

**Existem vários pacotes de atualização disponíveis para alguns dos Software Afetado. Eu devo instalar todas as atualizações listadas na tabela de Software Afetado para o software?**
Sim. Os clientes devem aplicar todas as atualizações oferecidas para o software instalado em seus sistemas.

**Devo instalar essas atualizações de segurança em uma determinada sequência?**
Não. As várias atualizações de um sistema podem ser aplicadas em qualquer sequência.

Informações sobre a vulnerabilidade
-----------------------------------

<span id="sectionToggle4"></span>
Vulnerabilidade de divulgação de informações do ASP.NET - CVE-2015-1648
-----------------------------------------------------------------------

Existe uma vulnerabilidade de divulgação não autorizada de informações no ASP.NET que é causada quando o ASP.NET manipula de maneira inadequada determinadas solicitações em sistemas que têm as mensagens de erro personalizadas desabilitadas. Um atacante que explorar com êxito a vulnerabilidade poderia visualizar partes do arquivo de configuração da web, o que pode expor informações confidenciais.

Para explorar esta vulnerabilidade, um atacante pode enviar uma solicitação da web especialmente criada para um servidor afetado com a intenção de produzir uma mensagem de erro que pode divulgar informações não autorizadas pertencentes a linha de fonte que originou a exceção. Finalmente, isto pode divulgar informações não autorizadas que não eram destinadas a serem acessadas. A atualização de segurança aborda a vulnerabilidade ao remover os detalhes do conteúdo do arquivo das mensagens de erro que estavam facilitando a divulgação não autorizada de informações.

Por padrão, os aplicativos do ASP.NET não estão expostos a esta vulnerabilidade porque eles são configurados para não exibir mensagens de erro detalhadas aos usuários remotos. Às vezes os desenvolvedores ativam as mensagens de erro detalhadas para reunir informações e depois esquecem de desativá-las, o que então expõe o aplicativo a esta vulnerabilidade.

        ```

    <customErrors mode="remoteOnly" />
        ```
        ```

    <customErrors mode="On" defaultRedirect="ErrorPage.htm" /> 
        ```
        ```

    <customErrors mode="off" />
        ```
Observe que mensagens de erro podem ser personalizadas com base no código de erro. Para obter mais informações, consulte [Elemento customErrors (Esquema de configurações ASP.NET)](https://msdn.microsoft.com/pt-br/library/h0hfz6fc(v=vs.85).aspx).

Como uma proteção adicional contra a desativação acidental de erros personalizados, as administrações de máquina podem ativar o modo de varejo. Consulte a [solução alternativa](#configretail) relacionada neste boletim para obter mais informações.

A Microsoft recebeu informações sobre esta vulnerabilidade por meio da divulgação coordenada de vulnerabilidades. Quando esse boletim de segurança foi originalmente emitido, a Microsoft ainda não tinha recebido informações que indicassem que a vulnerabilidade tinha sido usada de forma pública para atacar clientes.

### Fatores atenuantes

Os seguintes [fatores atenuantes](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) podem ser úteis no seu caso:

-   **Somente os servidores IIS que servem as mensagens de erro detalhadas são afetados; é improvável que os servidores de produção sejam afetados.**

### Soluções alternativas

<span id="configretail"></span>
As seguintes [soluções alternativas](https://technet.microsoft.com/pt-br/library/security/dn848375.aspx) podem ser úteis em seu caso:

-   **Configure o .NET no modo de varejo em todos os servidores da web**

    Nos arquivos machine.config de todos os servidores da web, adicione a configuração "&lt;deployment retail="true" /&gt;" à seção "system.web" da seção "configuration".

    Em sistemas de 32 bits, o machine.config é encontrado em %windir%\\Microsoft.NET\\Framework\\\[version\]\\config\\machine.config.

    Em sistemas de 64 bits, o machine.config é encontrado em %windir%\\Microsoft.NET\\Framework64\\\[version\]\\config\\machine.config.

    Para obter mais informações sobre esta contribuição, consulte [Elemento httpRuntime (Esquema de configurações ASP.NET)](https://msdn.microsoft.com/pt-br/library/vstudio/ms228298(v=vs.100).aspx).

    **Impacto da solução alternativa.** Todas as mensagens de erro detalhadas do ASP.NET de todos os sites em cada servidor serão suprimidas.

    **Como desfazer a solução alternativa. **

    Para desfazer a solução alternativa, remova a configuração que foi adicionada por este procedimento.

     

-   **Habilite erros personalizados para todos os sites**

    Nos arquivos web.config para todos os sites, certifique-se de que a configuração "customErrors" (na seção "system.web" da seção "configuration") não esteja definida para “off”.  Os valores de segurança são "on", "remoteonly" ou nenhuma configuração.

    Para obter mais informações sobre a configuração customErrors, consulte [Elemento httpRuntime (Esquema de configurações ASP.NET)](https://msdn.microsoft.com/pt-br/library/h0hfz6fc(v=vs.85).aspx).

    **Impacto da solução alternativa.** Todas as mensagens de erro detalhadas do ASP.NET de todos os sites serão suprimidas.

    **Como desfazer a solução alternativa. **

    Para desfazer a solução alternativa, reverta as configurações estabelecidas por este procedimento.
